# 5.1 Semiconductor Main Memory

หน่วยความจำหลักในอดีตใช้เทคโนโลยีแกนแม่เหล็กคล้ายวงแหวนที่เรียกว่า คอร์ ทำให้อดีตคนก็เรียกหน่วยความจำว่า คอร์  
และเมื่อเทคโนโลยีไมโครอิเล็กทรอนิกส์พัฒนา จึงเปลี่ยนมาใช้ชิปสารกึ่งตัวนำ (Semiconductor) แทน เนื่องจากมีประสิทธิภาพดีกว่าและเป็นมาตรฐานในปัจจุบัน


## Organization

หน่วยความจำแบบสารกึ่งตัวนำประกอบด้วยหน่วยย่อยที่เรียกว่า memory cell โดยแต่ละเซลล์มีคุณสมบัติหลักคือ

- มีสถานะได้ 2 ค่า โดยใช้แทนข้อมูลแบบไบนารี คือ 0,1
- สามารถเขียนข้อมูลเพื่อกำหนดสถานะได้
- สามารถอ่านข้อมูลเพื่อตรวจสอบสถานะได้

การทำงานของ memory cell โดยทั่วไปจะมีขั้ว (terminal) หลัก 3 ส่วน ได้แก่

- Select: ใช้เลือกเซลล์ที่จะอ่านหรือเขียนข้อมูล
- Control: ใช้กำหนดว่าจะ “อ่าน” หรือ “เขียน”
- Data: ใช้รับข้อมูลเข้า (ตอนเขียน) หรือส่งข้อมูลออก (ตอนอ่าน)


## DRAM AND SRAM(RAM)

หน่วยความจำทุกประเภทในส่วนนี้เป็นแบบ Random Access ซึ่งสามารถเข้าถึงข้อมูลแต่ละตำแหน่งได้โดยตรงผ่านวงจรระบุตำแหน่ง (addressing) โดยไม่ต้องเรียงลำดับก่อน


## DRAM and SRAM

หน่วยความจำทั้งหมดในบทนี้เป็นแบบ Random Access คือสามารถเข้าถึงข้อมูลแต่ละตำแหน่งได้โดยตรงผ่านวงจร addressing  

RAM (Random Access Memory) เป็นหน่วยความจำที่อ่านและเขียนข้อมูลได้รวดเร็วผ่านสัญญาณไฟฟ้า และเป็นแบบ volatile (ไฟดับข้อมูลหาย) จึงใช้เป็นหน่วยความจำชั่วคราว โดยแบ่งเป็น 2 ประเภทหลักคือ DRAM และ SRAM


### DRAM (Dynamic RAM)

- เก็บข้อมูลในรูปประจุไฟฟ้าในตัวเก็บประจุ (capacitor)
- ต้องมีการ refresh เป็นระยะ เพราะประจุไฟฟ้ารั่วได้
- ขณะอ่านข้อมูลจะทำให้ประจุหาย ต้องเขียนกลับใหม่
- โครงสร้างเรียบง่าย ขนาดเล็ก → ความจุสูง ราคาถูก
- ทำงานแบบกึ่งอนาล็อก (ใช้ระดับประจุตีความเป็น 0/1)


### SRAM (Static RAM)

- เก็บข้อมูลด้วยวงจร flip-flop (logic gate)
- ไม่ต้อง refresh ข้อมูล
- คงข้อมูลได้ตราบใดที่ยังมีไฟเลี้ยง
- เร็วกว่า DRAM แต่โครงสร้างซับซ้อนและขนาดใหญ่กว่า


### เปรียบเทียบ DRAM vs SRAM

- DRAM: ราคาถูก ความจุสูง แต่ช้ากว่า และต้อง refresh
- SRAM: เร็วกว่า แต่ราคาแพงและความจุต่ำกว่า
- ทั้งคู่เป็น volatile

การใช้งาน
- SRAM → ใช้เป็น Cache Memory
- DRAM → ใช้เป็น Main Memory


## Types of ROM

ROM (Read-Only Memory) เป็นหน่วยความจำแบบ ไม่ลบเลือน (nonvolatile) ข้อมูลถูกบันทึกถาวร ไม่สามารถเขียนใหม่ได้ และไม่ต้องใช้ไฟเลี้ยงในการเก็บข้อมูล  
เหมาะสำหรับเก็บโปรแกรมหรือข้อมูลที่ต้องใช้ประจำ เช่น system program หรือ function table

ข้อเสียของ ROM
- ต้นทุนเริ่มต้นสูง
- แก้ไขข้อมูลไม่ได้ (ถ้าผิดต้องทิ้งทั้งหมด)

ประเภทของ ROM

1. PROM (Programmable ROM)
   - เขียนข้อมูลได้ ครั้งเดียว
   - เขียนด้วยไฟฟ้าหลังจากผลิตเสร็จ
   - ยืดหยุ่นกว่า ROM ปกติ

2. EPROM (Erasable PROM)
   - ลบข้อมูลได้ด้วย แสง UV
   - ต้องลบทั้งชิปก่อนเขียนใหม่
   - ใช้เวลาลบนาน (ประมาณหลายสิบนาที)
   - เขียนซ้ำได้หลายครั้ง

3. EEPROM (Electrically Erasable PROM)
   - ลบและเขียนใหม่ได้ด้วยไฟฟ้า
   - แก้ไขข้อมูลเป็น ราย byte ได้
   - ไม่ต้องลบทั้งชิป
   - ช้ากว่า RAM และราคาสูง

4. Flash Memory
   - พัฒนามาจาก EEPROM
   - ลบข้อมูลได้เป็น block (เร็วกว่า EPROM มาก)
   - ความจุสูงและราคาถูกกว่า EEPROM
   - ใช้กันแพร่หลาย (เช่น USB, SSD)

สรุปภาพรวม
- ROM → ถาวร แก้ไม่ได้
- PROM → เขียนได้ครั้งเดียว
- EPROM → ลบด้วยแสง
- EEPROM → ลบด้วยไฟฟ้า (ยืดหยุ่นสูง)
- Flash → เร็ว + ใช้งานจริงมากที่สุด


## Chip Logic

หน่วยความจำแบบสารกึ่งตัวนำถูกบรรจุในรูปแบบ ชิป (chip) โดยภายในประกอบด้วยอาร์เรย์ของ memory cell  

การออกแบบชิปต้องมีการแลกเปลี่ยนระหว่าง ความเร็ว ความจุ และต้นทุน โดยปัจจัยสำคัญคือจำนวนบิตที่สามารถอ่าน/เขียนได้ในครั้งเดียว

รูปแบบการจัดองค์กรของชิป
- แบบอ่าน/เขียนเป็น คำ (word) เช่น หลายบิตต่อครั้ง
- แบบ 1 บิตต่อชิป (อ่าน/เขียนทีละบิต)

โครงสร้างการทำงาน (ตัวอย่าง DRAM)
- เซลล์ถูกจัดเป็นตาราง แถว (row) และคอลัมน์ (column)
- เส้นแนวนอน → ใช้เลือกแถว (Select)
- เส้นแนวตั้ง → ใช้รับ/ส่งข้อมูล (Data)

การระบุตำแหน่ง (Addressing)
- ใช้ address lines เพื่อเลือกตำแหน่งข้อมูล
- แบ่งเป็นเลือกแถว (row decoder) และ เลือกคอลัมน์ (column)
- ข้อมูลจะถูกอ่าน/เขียนผ่าน data lines


## Chip Packaging

ชิปหน่วยความจำถูกบรรจุในแพ็กเกจที่มีขา (pins) สำหรับเชื่อมต่อกับภายนอก

EPROM (1M × 8) มี 32 ขา ประกอบด้วย
- address 20 ขา
- data 8 ขา
- power (Vcc) และ ground (Vss)
- chip enable (CE) ใช้เลือกชิป
- program voltage (Vpp) ใช้ตอนเขียนข้อมูล

DRAM (4M × 4)
- data เป็น input/output
- มีขา WE และ OE สำหรับกำหนดอ่าน/เขียน
- ใช้ address แบบ multiplex (row/column) จึงใช้เพียง 11 address pins
- มีขา RAS และ CAS สำหรับเลือกแถวและคอลัมน์
- มีขา NC เพื่อให้จำนวนขาเป็นเลขคู่


## Module Organization

หากชิปหน่วยความจำเก็บข้อมูลได้เพียง 1 บิตต่อคำ (word) จำเป็นต้องใช้หลายชิปตามจำนวนบิตของคำ

ตัวอย่าง:
- หน่วยความจำ 256K × 8 bits ใช้ชิป 8 ตัว (แต่ละตัวเก็บ 1 บิต)
- ใช้ 18-bit address เพื่อระบุตำแหน่งข้อมูล และส่งไปยังทุกชิป

เมื่อต้องการความจุมากขึ้น:
- ต้องจัดชิปเป็น หลายชุด (array)
- เช่น 1M × 8 bits ใช้หลายกลุ่มของชิป

ใช้ address 20 บิต โดย
- 18 บิตแรก → ใช้เลือกตำแหน่งภายในชิป
- 2 บิตสุดท้าย → ใช้เลือกกลุ่มชิปผ่าน chip enable


## Interleaved Memory

หน่วยความจำหลักประกอบด้วยชิป DRAM หลายตัว ซึ่งสามารถรวมกันเป็น memory bank  

การจัดแบบ interleaved memory ทำให้แต่ละ bank ทำงานได้อย่างอิสระ และสามารถอ่านและเขียนพร้อมกันได้หลายคำสั่งตามจำนวน Bank และช่วยเพิ่มความเร็วในการเข้าถึงข้อมูล  

หากเก็บข้อมูลต่อเนื่องไว้คนละ bank จะช่วยให้การอ่านข้อมูลเป็นชุด (block) เร็วขึ้น

# บทที่ 5.2 เรื่อง Error Correction (การแก้ไขข้อผิดพลาดในหน่วยความจำ)

ทำไมหน่วยความจำถึงต้องมีการแก้ข้อผิดพลาด? ระบบหน่วยความจำเซมิคอนดักเตอร์ที่เราใช้กันนั้นไม่ได้สมบูรณ์แบบร้อยเปอร์เซ็นต์ครับ มันมีโอกาสที่จะเกิด "ข้อผิดพลาด" (Errors) ขึ้นได้เสมอ ซึ่งในโลกของคอมพิวเตอร์จะแบ่งข้อผิดพลาดนี้ออกเป็น 2 ระดับหลักๆ:

## 1. Hard failures (พังแบบถาวร):
อันนี้คือความเสียหายทางกายภาพของเซลล์หน่วยความจำเลยครับ ทำให้เซลล์นั้นไม่สามารถเก็บข้อมูลได้อีกต่อไป มันอาจจะค้างติดอยู่ที่ค่า 0 หรือ 1 ตลอดเวลา หรือไม่ก็เดี๋ยวเป็น 0 เดี๋ยวเป็น 1 แบบควบคุมไม่ได้ สาเหตุก็มักจะมาจากการเอาไปใช้งานในสภาพแวดล้อมที่โหดร้ายเกินไป มีความบกพร่องมาตั้งแต่ขั้นตอนการผลิตโรงงาน หรือแค่มันเสื่อมสภาพเพราะใช้งานมานาน

## 2. Soft errors (พังแบบชั่วคราว):
อันนี้จะต่างออกไปครับ เพราะตัวหน่วยความจำไม่ได้พัง แต่มีเหตุการณ์แบบสุ่มบางอย่างมาทำให้ค่าข้อมูลที่เก็บไว้มัน "เพี้ยน" หรือกลับด้านไป สาเหตุยอดฮิตมักเกิดจากระบบจ่ายไฟมีปัญหา หรืออาจจะเกิดจาก อนุภาคแอลฟา (Alpha particles) ที่แผ่มาจากสารกัมมันตรังสีที่ปะปนอยู่ในวัสดุเกือบทุกชนิดบนโลกไปชนเข้ากับเซลล์หน่วยความจำ

แน่นอนว่าทั้งสองแบบนี้ทำให้ข้อมูลเราเสียหาย ด้วยเหตุนี้ ระบบหน่วยความจำหลักสมัยใหม่จึงต้องมีวงจรตรรกะ (Logic) พิเศษที่คอยทำหน้าที่ "ตรวจจับ" และ "แก้ไข" ข้อผิดพลาดเหล่านี้ติดตั้งมาด้วยเสมอ

**ระบบนี้ทำงานอย่างไร? (กระบวนการตรวจจับและแก้ไข):**

* **ตอนเขียนข้อมูล:** ลองจินตนาการว่าข้อมูลที่เราต้องการจัดเก็บมีความยาว M บิต แทนที่เราจะเอาไปเก็บตรงๆ ระบบจะเอาข้อมูลนี้ไปผ่านเครื่องคิดเลข (ฟังก์ชัน f) เพื่อสร้าง "รหัสลับ หรือ Code" ความยาว K บิตขึ้นมาก่อน แล้วระบบก็จะแพ็กข้อมูลกับรหัสลับนี้ติดกันเพื่อเอาไปเก็บไว้ในหน่วยความจำ ดังนั้นขนาดของพื้นที่ที่เราใช้เก็บจริงๆ จะกลายเป็น M+K บิต ครับ
* **ตอนอ่านข้อมูล:** ความน่าสนใจมันอยู่ที่ "ตอนดึงข้อมูลมาใช้งาน" ครับ: เมื่อระบบอ่านข้อมูลออกมา มันจะเอาข้อมูล M บิตที่อ่านได้ ไปคำนวณสร้างรหัสลับชุดใหม่ขึ้นมาอีกรอบ แล้วเอารหัสใหม่นี้ไป "เทียบ" กับรหัส K บิตตัวเก่าที่เราดึงออกมาด้วย 

ซึ่งผลลัพธ์จะออกหน้าใดหน้าหนึ่งใน 3 กรณีนี้:
1. ไม่มีอะไรผิดพลาด: ถ้ารหัสตรงกันเป๊ะ แปลว่าข้อมูลปลอดภัยดี ระบบก็จะส่งข้อมูลนั้นไปให้ซีพียูใช้งานต่อเลย
2. มีข้อผิดพลาด แต่เอาอยู่ (แก้ไขได้): ระบบจะส่งข้อมูลพร้อมกับบิตที่มีปัญหาเข้าไปในเครื่องมือที่เรียกว่า ตัวแก้ไข (Corrector) เพื่อจัดการซ่อมแซมข้อมูลให้กลับมาถูกต้องสมบูรณ์ก่อนส่งออกไปใช้งาน
3. มีข้อผิดพลาด แต่พังเกินกว่าจะแก้ไหว: ระบบจะยกธงขาว และรายงานเตือนว่าพบข้อผิดพลาดรุนแรง

**รู้จักกับ "รหัสแฮมมิง (Hamming Code)" ฮีโร่ผู้หาจุดผิดพลาด:**
รหัสลับที่เราพูดถึงกัน หนึ่งในวิธีที่คลาสสิกและเข้าใจง่ายที่สุดคิดค้นโดย Richard Hamming ครับ วิธีการของเขาคือการเอารหัสเก่ากับรหัสใหม่มาเทียบกันทีละบิตด้วยตรรกะ Exclusive-OR (XOR) ผลลัพธ์ที่ได้จากการเทียบนี้เราจะเรียกว่า "ซินโดรม (Syndrome word)" ซึ่งจะมีความยาว K บิตเท่ากับตัวรหัสครับ ตัวเลขของ Syndrome นี่แหละครับที่เป็นกุญแจสำคัญที่ใช้บ่งบอกว่า "มีข้อผิดพลาดไหม" และ "ถ้ามี มันอยู่ตรงไหน"

* จำนวนของบิตเช็ค (K) จะต้องมีมากพอที่จะระบุตำแหน่งได้ครอบคลุมทั้งบิตข้อมูล (M) และตัวบิตเช็ค (K) เอง จึงเกิดเป็นสมการว่า $2^K - 1 \ge M + K$
* ตัวอย่างเช่น ถ้าเรามีข้อมูล 8 บิต (M=8) เราต้องสร้างรหัสเช็คอย่างน้อย 4 บิต (K=4) เพราะ $2^4 - 1$ เท่ากับ 15 ซึ่งมากกว่าหรือเท่ากับ 8 + 4 (12) นั่นเองครับ

**รหัสซินโดรม (Syndrome) ทำงานยังไง?:**
สมมติว่าเราใช้รหัสเช็ค 4 บิตสำหรับข้อมูล 8 บิต (รวมเป็น 12 บิต) ค่าซินโดรมที่จะออกมาสามารถตีความได้ดังนี้ครับ:

* **ถ้าซินโดรมเป็น 0000:** ดีเยี่ยม! ไม่มีข้อผิดพลาด
* **ถ้าซินโดรมมีเลข 1 โผล่มาแค่ตัวเดียว (เช่น 0001, 0010):** แบบนี้แปลว่าข้อมูลเรายังปลอดภัยดีครับ แต่ตัวที่พังคือ "บิตเช็ค" ซึ่งเราไม่ต้องไปสนใจแก้ข้อมูลอะไร
* **ถ้าซินโดรมมีเลข 1 มากกว่าหนึ่งตัว:** แปลว่า "ข้อมูลพังแล้ว" และความเจ๋งคือ ค่าของซินโดรม (เมื่อแปลงเป็นตัวเลข) จะคอยชี้เป้าให้เลยว่าข้อมูลบิตตำแหน่งที่เท่าไหร่ที่ผิดเพี้ยนไป ระบบก็จะทำการกลับค่า (Invert) จาก 0 เป็น 1 หรือ 1 เป็น 0 ในตำแหน่งนั้นๆ เพื่อให้ข้อมูลกลับมาถูกต้องเหมือนเดิม

ยกตัวอย่างเพื่อให้เห็นภาพ สมมติว่าตอนแรกเราเก็บข้อมูลโดยเรียงรหัสไว้ด้วยกันแล้ว และจู่ๆ วันดีคืนดี ข้อมูลใน "ตำแหน่งที่ 6" ดันพัง โดนรังสีชนจนเปลี่ยนจาก 0 เป็น 1 พอระบบเอาข้อมูลมาอ่าน มันจะคำนวณรหัสเทียบกันด้วยวิธี XOR ทันที ผลลัพธ์ของซินโดรมรอบนี้จะออกมาเป็นเลข '0110' ซึ่งเมื่อเรานำเลขไบนารี 0110 ไปแปลงเป็นเลขฐานสิบ มันก็คือเลข 6 นั่นเองครับ! ระบบจึงรู้ได้ทันทีเลยว่า "อ๋อ ข้อผิดพลาดอยู่ที่ตำแหน่งที่ 6 นี่เอง" แล้วระบบก็จัดการซ่อมแซมได้อย่างแม่นยำ

**ปัญหาที่ Hamming Code ทั่วไปรับมือไม่ได้ (และการอัปเกรดเป็น SEC-DED):**
รหัสที่เราคุยกันมาทั้งหมดนี้ มีชื่อเรียกทางเทคนิคว่า SEC (Single-Error-Correcting) คือมันเก่งมากในการแก้ไขข้อผิดพลาดถ้ามีข้อมูลพังแค่ "1 บิต" แต่ในความเป็นจริง ถ้ามีข้อผิดพลาดเกิดขึ้น "พร้อมกัน 2 บิต" ระบบ SEC ปกติจะเกิดอาการ "รวน" คาดเดาผิดพลาด และอาจจะดันไปแก้ไขบิตอื่นให้พังหนักกว่าเดิมเข้าไปอีก

เพื่ออุดช่องโหว่นี้ หน่วยความจำส่วนใหญ่ในปัจจุบันจึงนิยมใช้รหัสที่เรียกว่า SEC-DED (Single-Error-Correcting, Double-Error-Detecting) แปลตรงตัวคือ "แก้ไขได้ 1 บิต และสามารถตรวจจับได้หากพังพร้อมกัน 2 บิต" วิธีการคือเขาจะยอมเพิ่มบิตตรวจสอบ (Parity bit) เข้าไปอีก 1 บิตในระบบ เพื่อให้มันคอยเช็คผลรวมทั้งหมดอีกชั้นนึงว่ามีความผิดปกติซ้อนทับกันอยู่หรือไม่

**ข้อแลกเปลี่ยนของการมีระบบแก้ไขข้อผิดพลาด (Overhead):**
แน่นอนว่าของดีก็ต้องมีราคาที่ต้องจ่ายครับ การที่หน่วยความจำมีความน่าเชื่อถือและป้องกันตัวเองได้ ต้องแลกมากับความซับซ้อนและ "พื้นที่" ที่ต้องใช้เผื่อในการจัดเก็บรหัสลับเหล่านี้ ยกตัวอย่างเช่น ระบบของคอมพิวเตอร์ VAX ในอดีต ต้องใช้พื้นที่เพิ่มถึง 22% ในหน่วยความจำเพื่อเก็บรหัสพวกนี้ หรือแม้แต่ระบบหน่วยความจำ DRAM ในคอมพิวเตอร์ยุคปัจจุบัน ก็ยังต้องสูญเสียพื้นที่ไปราวๆ 7% ถึง 20% เพียงเพื่อเป็น Overhead สำหรับเตรียมไว้ใช้เก็บรหัสป้องกันข้อมูลเหล่านี้

## ขั้นตอนการคำนวณรหัสแฮมมิง (Hamming Code) เพื่อใช้ในการตรวจจับและแก้ไขข้อผิดพลาดแบบ 1 บิต สำหรับคำข้อมูล (Data word) ขนาด 8 บิต

## ขั้นที่ 1: การคำนวณหาจำนวนบิตตรวจสอบ (Check Bits)
ก่อนอื่นต้องหาว่าข้อมูลขนาด M บิต ต้องใช้บิตตรวจสอบ (K บิต) จำนวนเท่าใด โดยอ้างอิงจากอสมการ: 2^K - 1 >= M + K

**แทนค่าข้อมูลขนาด 8 บิต (M=8):**
* หากลองให้ K=3 จะได้ 2^3 - 1 >= 8 + 3 (หรือ 7 >= 11) ซึ่ง **ไม่เป็นจริง**
* หากลองให้ K=4 จะได้ 2^4 - 1 >= 8 + 4 (หรือ 15 >= 12) ซึ่ง **เป็นจริง**

ดังนั้น ข้อมูล 8 บิต จะต้องใช้บิตตรวจสอบจำนวน 4 บิต ทำให้ความยาวของคำที่ต้องจัดเก็บทั้งหมดคือ 12 บิต (8+4)

---

## ขั้นที่ 2: การจัดวางตำแหน่งของบิต (Layout)
เราจะนำข้อมูลและบิตตรวจสอบมาเรียงกันเป็นคำขนาด 12 บิต โดยมีกฎว่า **บิตตำแหน่งที่เป็นเลขยกกำลัง 2** (คือตำแหน่งที่ 1, 2, 4 และ 8) จะถูกกำหนดให้เป็นบิตตรวจสอบ (C) ส่วนตำแหน่งที่เหลือจะเป็นบิตข้อมูล (D) ดังนี้:

* ตำแหน่งที่ 1: C1
* ตำแหน่งที่ 2: C2
* ตำแหน่งที่ 3: D1
* ตำแหน่งที่ 4: C4
* ตำแหน่งที่ 5: D2
* ตำแหน่งที่ 6: D3
* ตำแหน่งที่ 7: D4
* ตำแหน่งที่ 8: C8
* ตำแหน่งที่ 9: D5
* ตำแหน่งที่ 10: D6
* ตำแหน่งที่ 11: D7
* ตำแหน่งที่ 12: D8

> **หลักการทำงาน:** บิตตรวจสอบแต่ละตัว จะทำการตรวจสอบบิตข้อมูลที่มี "เลข 1" ตรงกับตำแหน่งไบนารีของตัวมันเอง ยกตัวอย่างเช่น ตำแหน่งที่ 7 (ในเลขฐานสองคือ 0111) มีเลข 1 อยู่ในตำแหน่งที่ 1, 2 และ 4 ดังนั้นข้อมูลในตำแหน่งที่ 7 (D4) จะถูกนำไปคำนวณใน C1, C2 และ C4

---

## ขั้นที่ 3: การคำนวณค่าบิตตรวจสอบก่อนจัดเก็บ
สมมติว่าข้อมูล 8 บิตที่เราต้องการจัดเก็บคือ `00111001` (โดยให้ D1 อยู่ทางขวาสุด) จะได้ค่า:
D1=1, D2=0, D3=0, D4=1, D5=1, D6=1, D7=0, D8=0

ระบบจะคำนวณบิตตรวจสอบด้วยการทำ Exclusive-OR (⊕) ตามสมการต่อไปนี้:
* **C1** = D1 ⊕ D2 ⊕ D4 ⊕ D5 ⊕ D7 = 1 ⊕ 0 ⊕ 1 ⊕ 1 ⊕ 0 = **1**
* **C2** = D1 ⊕ D3 ⊕ D4 ⊕ D6 ⊕ D7 = 1 ⊕ 0 ⊕ 1 ⊕ 1 ⊕ 0 = **1**
* **C4** = D2 ⊕ D3 ⊕ D4 ⊕ D8 = 0 ⊕ 0 ⊕ 1 ⊕ 0 = **1**
* **C8** = D5 ⊕ D6 ⊕ D7 ⊕ D8 = 1 ⊕ 1 ⊕ 0 ⊕ 0 = **0**

ดังนั้น รหัสตรวจสอบ 4 บิต (C8 C4 C2 C1) ที่ได้คือ `0111` และคำทั้งหมด 12 บิตที่จะถูกจัดเก็บลงหน่วยความจำ (เรียงตามตำแหน่ง 12 ถึง 1) คือ `001101001111`

---

## ขั้นที่ 4: เมื่อเกิดข้อผิดพลาดขึ้นในหน่วยความจำ
สมมติว่าเกิดข้อผิดพลาดขึ้นที่ **บิตข้อมูล D3 (ตำแหน่งที่ 6)** ทำให้ค่าถูกเปลี่ยนจาก 0 เป็น 1 เมื่อระบบอ่านข้อมูลนี้ออกมา ข้อมูลที่ดึงออกมาได้จะกลายเป็น `001101101111` (ตำแหน่งที่ 6 เพี้ยนไป)

---

## ขั้นที่ 5: การอ่านและคำนวณบิตตรวจสอบชุดใหม่
เมื่ออ่านข้อมูลออกมา ระบบจะใช้ข้อมูลชุดใหม่ (ที่บิต D3 กลายเป็น 1 ไปแล้ว) มาคำนวณหาค่าบิตตรวจสอบอีกครั้งด้วยสมการเดิม:
* **C1 ใหม่** = 1 ⊕ 0 ⊕ 1 ⊕ 1 ⊕ 0 = **1**
* **C2 ใหม่** = 1 ⊕ 1 ⊕ 1 ⊕ 1 ⊕ 0 = **0** *(ผลลัพธ์เปลี่ยนเพราะ D3 เปลี่ยน)*
* **C4 ใหม่** = 0 ⊕ 1 ⊕ 1 ⊕ 0 = **0** *(ผลลัพธ์เปลี่ยนเพราะ D3 เปลี่ยน)*
* **C8 ใหม่** = 1 ⊕ 1 ⊕ 0 ⊕ 0 = **0**

บิตตรวจสอบชุดใหม่ที่คำนวณได้คือ `0001`

---

## ขั้นที่ 6: การคำนวณคำซินโดรม (Syndrome Word) และระบุตำแหน่งที่ผิดพลาด
ระบบจะนำ **รหัสชุดเก่า** มาทำ Exclusive-OR (⊕) กับ **รหัสชุดใหม่** เพื่อหาคำซินโดรม:

* รหัสเก่า:  `0 1 1 1`
* รหัสใหม่:  `0 0 0 1` (⊕)
* **ผลลัพธ์ซินโดรม:** `0 1 1 0`

ผลลัพธ์ซินโดรมที่ได้คือ `0110` ซึ่งเมื่อแปลงเป็นเลขฐานสิบ จะมีค่าเท่ากับ **6** ค่าซินโดรมนี้เป็นตัวระบุอย่างชัดเจนว่า **ข้อผิดพลาดเกิดขึ้นที่ตำแหน่งที่ 6** (ซึ่งก็คือบิตข้อมูล D3) จากนั้นตัวแก้ไข (Corrector) ของระบบจะทำการกลับค่า (Invert) ข้อมูลในตำแหน่งที่ 6 จาก 1 ให้กลับเป็น 0 เพื่อซ่อมแซมข้อมูลให้ถูกต้องก่อนส่งออกไปใช้งาน

---

# 5.3 DDR DRAM

### 1. ปัญหาของ DRAM แบบดั้งเดิม
* **คอขวด (Bottleneck)**: ความเร็ว RAM ตาม CPU ไม่ทัน
* **อซิงโครนัส**: ทำงานไม่ตรงจังหวะสัญญาณนาฬิกา ทำให้ CPU ต้องหยุดรอ (**Wait States**)

### 2. Synchronous DRAM (SDRAM)
* **จุดเด่น**: ทำงานสัมพันธ์กับ **Clock** ระบบ รับส่งข้อมูลเต็มความเร็วบัส
* **Burst Mode**: ดึงข้อมูลเป็นชุดต่อเนื่องได้รวดเร็ว (ไม่ต้องส่ง Address ซ้ำ)
* **Multiple Banks**: แบ่งพื้นที่ภายในเป็นหลายส่วน เพื่อให้ทำงานขนานกันได้

### 3. Double Data Rate (DDR SDRAM)
พัฒนาจาก SDRAM ให้เร็วขึ้น 2 เท่าด้วยเทคนิค:
* **Double Data Rate**: รับส่งข้อมูลทั้ง **ขาขึ้นและขาลง** ของสัญญาณนาฬิกา
* **Prefetch Buffer**: ระบบดึงข้อมูลล่วงหน้าจากแกนหลัก (Core) มาจ่อรอที่ส่วนเชื่อมต่อ (Interface)

### 4. วิวัฒนาการแต่ละยุค (DDR1 - DDR4)
เน้นเพิ่มขนาด **Prefetch** เพื่อเร่งความเร็ว ในขณะที่แกน RAM ยังทำงานความเร็วเท่าเดิม:
* **DDR1**: Prefetch **2-bit**
* **DDR2**: Prefetch **4-bit** (เร็วขึ้น 4 เท่าของแกน)
* **DDR3**: Prefetch **8-bit** (เร็วขึ้น 8 เท่าของแกน)
* **DDR4**: ใช้ Prefetch **8-bit** เท่าเดิม แต่เพิ่มระบบ **Bank Groups** เพื่อส่งข้อมูลขนานกันได้มากขึ้น และลดพลังงานเหลือ **1.2V**

---

## Synchronous DRAM (SDRAM)
1. นิยามและความแตกต่าง
* **SDRAM**: ทำงานสัมพันธ์กับ สัญญาณนาฬิกาภายนอก (External Clock) ทำให้รับส่งข้อมูลได้เต็มความเร็วบัสโดยไม่มีสถานะรอคอย (Wait States)

* **DRAM เดิม**(Asynchronous): ทำงานไม่ตรงจังหวะสัญญาณนาฬิกา ทำให้ตัวประมวลผลต้องเสียเวลารอในระหว่างกระบวนการภายในของหน่วยความจำ

2. ปัญหาของ DRAM แบบดั้งเดิม
* ในขณะที่ DRAM กำลังค้นหาข้อมูล (Access Time) เช่น การกระตุ้นแถว/คอลัมน์ หรือการส่งข้อมูลผ่านบัฟเฟอร์ ตัวประมวลผลต้องหยุดรอ (Wait) ส่งผลให้ประสิทธิภาพโดยรวมของระบบช้าลง

3. กลไกการทำงานของ SDRAM
* การควบคุม: ใช้ระบบสัญญาณนาฬิกาควบคุมการเข้าและออกของข้อมูล
* การล็อคสัญญาณ (Latch): เมื่อตัวประมวลผลส่งคำสั่งและที่อยู่ข้อมูล SDRAM จะทำการล็อคค่าเหล่านั้นไว้ทันที
* การประมวลผลขนาน: SDRAM จะตอบกลับตามจำนวนรอบสัญญาณนาฬิกาที่กำหนดไว้ล่วงหน้า ทำให้ระหว่างที่รอข้อมูล ตัวประมวลผลสามารถไปทำงานอื่นได้ ไม่ต้องหยุดรอเหมือนระบบเก่า

4. องค์ประกอบทางกายภาพ
* ชิป SDRAM มาตรฐาน (เช่น 256-Mb) มีโครงสร้างตรรกะภายในที่ซับซ้อน และมีการกำหนดหน้าที่ของพิน (Pin Assignments) ที่ชัดเจน เพื่อให้การสื่อสารสอดคล้องกับจังหวะสัญญาณนาฬิกาอย่างแม่นยำ

## SDRAM(Pin Assignments)

พินของ SDRAM แบ่งเป็น 4 กลุ่มหลัก ทำหน้าที่ประสานงานตามจังหวะ **Clock (CLK)** ดังนี้:

### 1. กลุ่มระบุตำแหน่ง (Address)
* **A0 - An**: ระบุแถว (**Row**) และคอลัมน์ (**Column**) ของข้อมูล
* **BA0, BA1**: เลือก **Bank** (ธนาคารข้อมูล) ภายในชิป

### 2. กลุ่มคำสั่งและควบคุม (Control)
* **CLK**: หัวใจหลัก กำหนดจังหวะการทำงาน (อ่านสัญญาณที่ขอบขาขึ้น)
* **CKE**: เปิด/ปิดการทำงานของชิป (Low = โหมดประหยัดไฟ)
* **$\overline{CS}$**: เลือกใช้งานชิปตัวนั้นๆ (Chip Select)
* **$\overline{RAS}, \overline{CAS}, \overline{WE}$**: ทำงานร่วมกันเพื่อสั่งการ (เช่น สั่งเปิดแถว, สั่งอ่าน หรือสั่งเขียน)

### 3. กลุ่มรับ-ส่งข้อมูล (Data)
* **DQ0 - DQn**: บัสข้อมูล (Data I/O) ใช้ทั้งรับเข้าและส่งออก
* **DQM**: ตัวควบคุมข้อมูล (Data Mask) ถ้าเป็น High จะระงับการส่งข้อมูลทันที

### 4. กลุ่มพลังงาน (Power)
* **$V_{CC}$ / $V_{SS}$**: ไฟเลี้ยงและกราวด์หลักของวงจรภายใน
* **$V_{CCQ}$ / $V_{SSQ}$**: ไฟเลี้ยงแยกส่วน **Input/Output** เพื่อลดสัญญาณรบกวน (Noise)

# คุณสมบัติขั้นสูงของ SDRAM (Advanced Features)
### 1. โหมดส่งข้อมูลต่อเนื่อง (Burst Mode)
* **กลไก**: ระบุตำแหน่งแรกเพียงครั้งเดียว ข้อมูลถัดไปในแถวเดียวกันจะถูกส่งออกมาทันทีตามจังหวะสัญญาณนาฬิกา
* **ข้อดี**: ลดเวลาการส่ง Address และการ Precharge ใหม่ทุกรอบ ทำให้รับ-ส่งข้อมูลบล็อกใหญ่ได้เร็วมาก

### 2. สถาปัตยกรรมหลายธนาคาร (Multiple-Bank)
* **การทำงานขนาน**: แบ่งพื้นที่ในชิปเป็นหลาย **Banks** อิสระ
* **จุดเด่น**: สามารถ "เตรียมข้อมูล" ใน Bank หนึ่ง ขณะที่ "ส่งข้อมูล" จากอีก Bank หนึ่งได้ (ลด Idle Time)

### 3. เรจิสเตอร์โหมด (Mode Register)
ใช้ปรับแต่งค่าการทำงานภายในชิป:
* **Burst Length**: กำหนดจำนวนข้อมูลที่จะส่งต่อ 1 คำสั่ง (เช่น 1, 2, 4, 8)
* **CAS Latency (CL)**: กำหนดรอบสัญญาณนาฬิกาที่ต้องรอ จนกว่าข้อมูลจะเริ่มไหลออก

### 4. ตัวอย่างการทำงาน (Burst=4, Latency=2)
1. **Command**: ส่งสัญญาณอ่าน (CS/CAS = Low) พร้อมระบุตำแหน่งคอลัมน์
2. **Waiting**: ระบบรอ 2 รอบสัญญาณนาฬิกา (ตามค่า Latency)
3. **Data Out**: ข้อมูล 4 ชุดถูกยิงออกมาต่อเนื่องกันจนครบ (1 ชุดต่อ 1 รอบ)

### 1. โครงสร้างภายใน (Internal Structure)
SDRAM 256-Mb แบ่งเป็น **4 Banks** เพื่อการทำงานขนาน:
* **Command Decoder**: รับคำสั่ง (Read/Write) จาก CPU
* **Mode Register**: ตั้งค่าสำคัญ เช่น **Burst Length** และ **Latency**
* **Address Latch**: ล็อกตำแหน่ง "แถว" และ "คอลัมน์"
* **Bank Control**: ควบคุมการเปิด/ปิด Bank เพื่อเตรียมข้อมูลล่วงหน้า

### 2. ลำดับเวลาการอ่าน (Read Timing)
*กรณี Burst=4, Latency=2*
1. **T0 (Read)**: ส่งคำสั่งอ่านที่ตำแหน่ง A
2. **T1 - T2 (Wait)**: ช่วงรอ 2 จังหวะ (Latency) ชิปกำลังค้นหาข้อมูล
3. **T3 (Data Start)**: ข้อมูลตัวแรก ($A_0$) เริ่มไหลออก
4. **T4 - T6 (Burst)**: ข้อมูลที่เหลือ ($A_1, A_2, A_3$) ไหลออกต่อเนื่องจนครบ

# DDR DRAM: วิวัฒนาการก้าวข้ามขีดจำกัด SDRAM
JEDEC พัฒนา **DDR (Double Data Rate)** เพื่อเพิ่มความเร็วในการรับ-ส่งข้อมูลแบบก้าวกระโดดผ่าน 3 กลยุทธ์หลัก:

### 1. วิธีเพิ่มความเร็ว (3 Key Strategies)
* **Double Data Rate**: ส่งข้อมูลทั้ง **"ขาขึ้น"** และ **"ขาลง"** ของสัญญาณนาฬิกา (เร็วขึ้น 2 เท่าทันที)
* **Higher Clock Rate**: เพิ่มความถี่สัญญาณนาฬิกาบนบัส (Bus)
* **Buffering Scheme**: ปรับปรุงระบบบัฟเฟอร์ให้จัดการข้อมูลได้ประสิทธิภาพสูงขึ้น

### 2. Prefetch Buffer (หัวใจของความเร็ว)
เนื่องจาก **Core** (แกนหลัก RAM) ทำงานช้ากว่า **Interface** (ส่วนเชื่อมต่อ) จึงต้องใช้ระบบดึงข้อมูลล่วงหน้า:
* **หลักการ**: ดึงข้อมูลจากแกนหลักออกมาหลายชุดพร้อมกันแบบขนาน (**Parallel**) แล้วนำมาเรียงลำดับส่งออกไปทีละตัวด้วยความเร็วสูง (**Serializing**) ผ่านตัว **MUX**

### 3. วิวัฒนาการแต่ละเจเนอเรชัน (Prefetch Size)
เมื่อความเร็วแกนหลักเพิ่มได้ยาก จึงใช้วิธี **"ขยายขนาดบัฟเฟอร์"** ในแต่ละยุคแทน:
* **DDR1**: 2-bit Prefetch (เร็วขึ้น 2 เท่าของความเร็วแกน)
* **DDR2**: 4-bit Prefetch (เร็วขึ้น 4 เท่าของความเร็วแกน)
* **DDR3**: 8-bit Prefetch (เร็วขึ้น 8 เท่าของความเร็วแกน)

# นวัตกรรม Bank Group ใน DDR4
### 1. ข้อเสียของระบบ Prefetch (The Downside)
* **ปัญหา**: ยิ่งเพิ่มขนาด **Prefetch** (เช่น 8-bit ใน DDR3) จะบังคับให้การส่งข้อมูลขั้นต่ำ (**Minimum Burst Length**) ยาวขึ้นตามไปด้วย
* **ผลกระทบ**: หากต้องการข้อมูลเพียง 4 คำ แต่ระบบบังคับส่ง 8 คำ จะเกิดข้อมูลส่วนเกินที่ไม่ได้ใช้ ทำให้ประสิทธิภาพลดลง

### 2. แนวคิดใหม่ใน DDR4: Bank Group
เพื่อเลี่ยงการใช้ Prefetch 16-bit ที่ควบคุมยาก วิศวกรจึงใช้เทคนิค **Bank Group** แทน:
* **แยกอิสระ**: แบ่งกลุ่มธนาคารข้อมูลออกเป็นชุดๆ (**Entities**) ที่ไม่เกี่ยวข้องกัน
* **ทำงานขนาน**: กลุ่มหนึ่งทำงานได้โดยไม่รบกวนอีกกลุ่ม ทำให้สั่งดึงข้อมูล 8-bit จากสองกลุ่มพร้อมกันได้

### 3. ผลลัพธ์ที่ได้
* **เร็วแต่ไม่เทอะทะ**: DDR4 ใช้ Prefetch 8-bit เท่าเดิม (ข้อมูลไม่ยาวเกินไป) แต่ได้ความเร็วสูงขึ้นจากการทำงานขนาน
* **ขีดความสามารถ**: รองรับ **Bank Group** ได้สูงสุด 4 กลุ่ม ทำให้ได้ประสิทธิภาพเสมือนมี Prefetch ขนาดใหญ่ขึ้นมากโดยไม่มีข้อเสียเรื่องข้อมูลส่วนเกิน

# 5.4 หน่วยความจำแบบแฟลช (Flash Memory)

## นิยามและลักษณะเฉพาะทางกายภาพ
หน่วยความจำแฟลช (Flash Memory) เป็นรูปแบบหนึ่งของหน่วยความจำสารกึ่งตัวนำ (Semiconductor Memory) ที่มีความสำคัญอย่างยิ่งในระบบคอมพิวเตอร์ปัจจุบัน โดยมีการประยุกต์ใช้งานอย่างกว้างขวางทั้งในลักษณะของหน่วยความจำภายใน (Internal Memory) และอุปกรณ์เก็บข้อมูลภายนอก (External Memory) ในเชิงประวัติศาสตร์ เทคโนโลยีนี้ถูกนำเสนอเป็นครั้งแรกในช่วงกลางทศวรรษที่ 1980 โดยมีคุณสมบัติที่อยู่กึ่งกลางระหว่างหน่วยความจำประเภท EPROM และ EEPROM ทั้งในด้านต้นทุนการผลิตและประสิทธิภาพการทำงาน

### ประสิทธิภาพในการจัดการข้อมูล
จุดเด่นที่สำคัญของหน่วยความจำแฟลชคือการใช้เทคโนโลยีการลบข้อมูลด้วยสัญญาณไฟฟ้า (Electrical Erasing) เช่นเดียวกับ EEPROM แต่มีความเร็วที่เหนือกว่าอย่างเห็นได้ชัด โดยสามารถลบข้อมูลในหน่วยความจำทั้งหมดได้ภายในระยะเวลาเพียงไม่กี่วินาที ซึ่งรวดเร็วกว่าเทคโนโลยี EPROM แบบดั้งเดิมมาก นอกจากนี้ ระบบยังได้รับการออกแบบให้มีความยืดหยุ่นสูง โดยอนุญาตให้ผู้ใช้เลือกปฏิบัติการลบข้อมูลเฉพาะส่วนหรือ "บล็อก" (Blocks) ได้ แทนที่จะต้องลบข้อมูลทั้งชิป (Chip) ในคราวเดียว

### โครงสร้างและความหนาแน่นของข้อมูล
ที่มาของชื่อ "Flash" นั้นมีรากฐานมาจากการจัดระเบียบโครงสร้างภายในไมโครชิป ที่ยอมรับให้กลุ่มของเซลล์หน่วยความจำสามารถถูกลบสถานะได้พร้อมกันในทันทีเสมือนการสาดแสงแฟลช อย่างไรก็ตาม หน่วยความจำแฟลชมีข้อจำกัดทางเทคนิคคือไม่รองรับการลบข้อมูลในระดับไบต์ (Byte-level erasure) แต่เพื่อเป็นการชดเชย เทคโนโลยีนี้ใช้โครงสร้างทรานซิสเตอร์เพียงหนึ่งตัวต่อหนึ่งบิตข้อมูล ส่งผลให้มีความหนาแน่นของข้อมูล (Density) สูงใกล้เคียงกับ EPROM และสูงกว่าหน่วยความจำประเภท EEPROM ในสัดส่วนพื้นที่ที่เท่ากัน

## หลักการและกลไกการทำงาน (Operation)
การทำงานของหน่วยความจำแฟลชอาศัยหลักการควบคุมประจุไฟฟ้าภายในโครงสร้างระดับไมโคร เพื่อใช้เป็นตัวแทนของข้อมูลในเชิงตรรกะ โดยมีการเปรียบเทียบกลไกพื้นฐานกับทรานซิสเตอร์ทั่วไปเพื่อให้เห็นภาพการทำงานที่ชัดเจนขึ้น

### โครงสร้างพื้นฐานของทรานซิสเตอร์
ในเชิงวิศวกรรมไฟฟ้า ทรานซิสเตอร์มาตรฐานจะใช้คุณสมบัติของสารกึ่งตัวนำในการทำหน้าที่เป็นสวิตช์ควบคุม โดยเมื่อมีการจ่ายแรงดันไฟฟ้าจำนวนเล็กน้อยไปที่ **เกต (Gate)** จะสามารถควบคุมการไหลของกระแสไฟฟ้าปริมาณมหาศาลระหว่าง **ซอร์ส (Source)** และ **เดรน (Drain)** ได้ ซึ่งถือเป็นกลไกพื้นฐานของการประมวลผลทางไฟฟ้า

### นวัตกรรมของหน่วยความจำแฟลช
สำหรับเซลล์ของหน่วยความจำแฟลช ได้มีการเพิ่มส่วนประกอบพิเศษที่เรียกว่า **เกตลอย (Floating Gate)** เข้าไปในโครงสร้างทรานซิสเตอร์ โดยเกตลอยนี้จะถูกหุ้มด้วยชั้นออกไซด์บางๆ ที่ทำหน้าที่เป็นฉนวนไฟฟ้าโดยสมบูรณ์ ทำให้มันสามารถกักเก็บประจุได้แม้ไม่มีไฟฟ้าเลี้ยงระบบ

* **สถานะลอจิก 1 (Binary 1):** ในสภาวะเริ่มต้นที่เกตลอยไม่มีประจุไฟฟ้าสะสมอยู่ มันจะไม่ส่งผลกระทบใดๆ ต่อการทำงานของทรานซิสเตอร์ ระบบจะถือว่าเซลล์ในสถานะนี้แทนค่าเลขฐานสองคือ 1
* **สถานะลอจิก 0 (Binary 0):** เมื่อมีการจ่ายแรงดันไฟฟ้าสูงข้ามชั้นออกไซด์ อิเล็กตรอนจะเกิดปรากฏการณ์ทางควอนตัมที่เรียกว่า "Tunneling" ทะลุผ่านชั้นฉนวนเข้าไปติดค้างอยู่ในเกตลอย แม้จะมีการตัดแหล่งจ่ายไฟออกไปแต่อิเล็กตรอนจะยังคงถูกขังอยู่ภายใน ทำให้ทรานซิสเตอร์เปลี่ยนสถานะไป และระบบจะถือว่าเซลล์นี้แทนค่าเลขฐานสองคือ 0

### การอ่านข้อมูลและการคืนสภาพ
การตรวจสอบสถานะหรือการอ่านข้อมูล (Reading) จะกระทำผ่านวงจรภายนอกที่ทดสอบว่าทรานซิสเตอร์ยังสามารถทำงานได้ตามปกติหรือไม่ และหากต้องการลบข้อมูลเพื่อเริ่มต้นใหม่ จะมีการจ่ายแรงดันไฟฟ้าสูงในทิศทางตรงกันข้ามเพื่อดึงอิเล็กตรอนออกจากเกตลอย ส่งผลให้เซลล์กลับคืนสู่สภาวะลอจิก 1 พร้อมสำหรับการบันทึกข้อมูลใหม่อีกครั้ง

---

## กลไกการทำงานของหน่วยความจำแฟลช (Operation of Flash Memory)

การทำความเข้าใจกลไกของหน่วยความจำแฟลช จำเป็นต้องพิจารณาเปรียบเทียบกับโครงสร้างพื้นฐานของทรานซิสเตอร์ เพื่อให้เห็นถึงนวัตกรรมในการกักเก็บประจุที่ทำให้ข้อมูลคงอยู่ได้แม้ไม่มีกระแสไฟฟ้า

### โครงสร้างและการทำงานของทรานซิสเตอร์พื้นฐาน
โดยพื้นฐานแล้ว ทรานซิสเตอร์ (Transistor) จะใช้ประโยชน์จากคุณสมบัติของสารกึ่งตัวนำในการควบคุมทิศทางไหลของไฟฟ้า เมื่อมีการจ่ายแรงดันไฟฟ้าขนาดเล็กเข้าไปที่ส่วน **เกต (Gate)** แรงดันนี้จะทำหน้าที่เป็นตัวเปิดช่องทางเพื่อให้กระแสไฟฟ้าปริมาณมากสามารถไหลผ่านระหว่าง **ซอร์ส (Source)** และ **เดรน (Drain)** ได้ ซึ่งเป็นกลไกสวิตชิ่ง (Switching) ที่ใช้ในวงจรประมวลผลทั่วไป

### โครงสร้างเกตลอย (Floating Gate) ในเซลล์หน่วยความจำ
สำหรับเซลล์ของหน่วยความจำแฟลช ได้มีการปรับปรุงโครงสร้างโดยการเพิ่มเกตตัวที่สองที่เรียกว่า **เกตลอย (Floating Gate)** เข้าไป ซึ่งเกตชนิดนี้ถูกตั้งชื่อตามลักษณะการติดตั้งที่แยกส่วนออกมา และถูกหุ้มด้วยชั้นออกไซด์บางๆ (Thin Oxide Layer) ที่ทำหน้าที่เป็นฉนวนไฟฟ้าโดยสมบูรณ์

### กระบวนการเปลี่ยนแปลงสถานะลอจิก
การจัดเก็บข้อมูลในหน่วยความจำแฟลชอาศัยการเปลี่ยนแปลงทางฟิสิกส์ภายในเซลล์เพื่อแทนค่าเลขฐานสอง (Binary) ดังนี้:

* **สถานะลอจิก 1 (Binary 1):** ในสภาวะเริ่มต้นที่เกตลอยยังไม่มีประจุไฟฟ้าสะสมอยู่ มันจะไม่มีผลต่อการทำงานของทรานซิสเตอร์ ทำให้กระแสไฟฟ้าไหลผ่านได้ตามปกติ ซึ่งระบบจะตีความเป็นค่าลอจิก 1
* **สถานะลอจิก 0 (Binary 0):** เมื่อมีการจ่ายแรงดันไฟฟ้าสูงข้ามชั้นออกไซด์ จะทำให้อิเล็กตรอนเกิดการเคลื่อนที่ผ่านฉนวนด้วยปรากฏการณ์ **Tunneling** และเข้าไปติดค้าง (Trapped) อยู่ภายในเกตลอย ประจุเหล่านี้จะยังคงอยู่แม้จะมีการตัดแหล่งจ่ายไฟออกไป ทำให้ทรานซิสเตอร์เปลี่ยนสภาวะการทำงาน ซึ่งระบบจะตีความเป็นค่าลอจิก 0

### การอ่านและการลบข้อมูล
ในกระบวนการอ่านข้อมูล (Read Operation) วงจรภายนอกจะทำการตรวจสอบว่าทรานซิสเตอร์ยังคงสามารถนำไฟฟ้าได้ตามปกติหรือไม่เพื่อระบุค่าสถานะ และเมื่อต้องการลบข้อมูล (Erase Operation) จะมีการจ่ายแรงดันไฟฟ้าสูงในทิศทางตรงกันข้ามเพื่อดึงอิเล็กตรอนออกจากเกตลอย ส่งผลให้เซลล์กลับคืนสู่สถานะลอจิก 1 และพร้อมสำหรับการบันทึกข้อมูลในรอบถัดไป

---

## ประเภทของหน่วยความจำแฟลช: NOR และ NAND

คุณสมบัติที่สำคัญประการหนึ่งของหน่วยความจำแฟลชคือ **ความคงทนของข้อมูล (Persistence)** ซึ่งหมายถึงความสามารถในการรักษาข้อมูลไว้ได้แม้ไม่มีกระแสไฟฟ้าหล่อเลี้ยง คุณลักษณะนี้ทำให้แฟลชกลายเป็นตัวเลือกหลักสำหรับทั้งหน่วยความจำสำรองภายนอก และใช้เป็นทางเลือกแทนหน่วยความจำเข้าถึงโดยสุ่ม (RAM) ในคอมพิวเตอร์สมัยใหม่ โดยสามารถแบ่งเทคโนโลยีออกเป็น 2 ประเภทหลักตามลักษณะโครงสร้างวงจร ดังนี้:

### หน่วยความจำแฟลชแบบ NOR (NOR Flash)
หน่วยความจำประเภทนี้มีโครงสร้างการเชื่อมต่อเซลล์หน่วยความจำแบบขนาน (Parallel) เข้ากับสายบิต (Bit lines) ซึ่งมีลักษณะการทำงานคล้ายกับลอจิกเกตชนิด NOR

* **หน่วยการเข้าถึง:** ใช้ "บิต" เป็นหน่วยพื้นฐานในการเข้าถึงข้อมูล
* **จุดเด่น:** เนื่องจากแต่ละเซลล์เชื่อมต่อแบบขนาน ทำให้ระบบสามารถอ่าน เขียน หรือลบข้อมูลในแต่ละเซลล์แยกกันได้อย่างอิสระ
* **กลไกไฟฟ้า:** หากเซลล์หน่วยความจำตัวใดตัวหนึ่งถูกเปิดการทำงานโดยสายคำ (Word line) ที่สอดคล้องกัน สายบิตจะมีสถานะเป็น Low ทันที

### หน่วยความจำแฟลชแบบ NAND (NAND Flash)
ในทางตรงกันข้าม NAND Flash จะจัดระเบียบโครงสร้างในรูปแบบของแถวทรานซิสเตอร์ (Transistor arrays) ที่เชื่อมต่อกันแบบอนุกรม (Series) โดยปกติจะเรียงต่อกันชุดละ 16 หรือ 32 ตัว

* **ลักษณะโครงสร้าง:** การเชื่อมต่อแบบอนุกรมนี้ทำให้มีกลไกการทำงานคล้ายกับลอจิกเกตชนิด NAND
* **กลไกไฟฟ้า:** สายบิตจะมีสถานะเป็น Low ก็ต่อเมื่อทรานซิสเตอร์ "ทุกตัว" ในสายคำนั้นถูกเปิดการทำงานพร้อมกันเท่านั้น
* **การใช้งาน:** ด้วยโครงสร้างแบบอนุกรมนี้เองที่ทำให้ NAND Flash สามารถทำความหนาแน่นของข้อมูลได้สูงกว่าในราคาที่ประหยัดกว่า จึงเป็นที่นิยมในอุปกรณ์เก็บข้อมูลความจุสูง

---

## การเปรียบเทียบคุณสมบัติ
แม้ว่าค่าตัวเลขทางเทคนิคของทั้ง NOR และ NAND จะมีการพัฒนาให้ดีขึ้นอย่างต่อเนื่องในทุกปี แต่ความแตกต่างเชิงเปรียบเทียบระหว่างทั้งสองประเภทนับว่าค่อนข้างคงที่ ซึ่งสามารถสรุปข้อแตกต่างสำคัญผ่านกราฟวิเคราะห์ (Kiviat graphs) เพื่อเปรียบเทียบในมิติต่างๆ เช่น ความเร็วในการอ่าน, ความเร็วในการเขียน, ความทนทาน และต้นทุนต่อความจุ

---

## โครงสร้างและการเปรียบเทียบประสิทธิภาพ

จากความแตกต่างในระดับวงจรไฟฟ้าของหน่วยความจำแฟลชทั้งสองประเภท ส่งผลให้คุณลักษณะเชิงประสิทธิภาพมีความแตกต่างกันอย่างมีนัยสำคัญ ซึ่งสามารถอธิบายผ่านสัญลักษณ์ทางตรรกะและเครื่องมือวิเคราะห์เชิงภาพได้ดังนี้

### การตีความทางตรรกะ (Logic Interpretation)
ในแผนผังโครงสร้างของหน่วยความจำแฟลช (Figure 5.16) การทำงานของเซลล์จะสัมพันธ์กับลอจิกเกตพื้นฐาน:
* **สัญญาณนิเสธ (Signal Negation):** สัญลักษณ์วงจรที่แสดงในแผนผังระบุถึงการกลับค่าสัญญาณไฟฟ้า ซึ่งเป็นหัวใจสำคัญที่ทำให้โครงสร้างแบบขนานทำหน้าที่เหมือน **NOR Gate** และโครงสร้างแบบอนุกรมทำหน้าที่เหมือน **NAND Gate**
* **ความสัมพันธ์กับสายบิต (Bit Line):** ลักษณะการเชื่อมต่อทรานซิสเตอร์ส่งผลโดยตรงต่อวิธีที่ระบบตรวจจับข้อมูล (Sense) ว่าเป็นลอจิก 0 หรือ 1 ตามเงื่อนไขทางไฟฟ้าที่ได้กล่าวไปในหัวข้อก่อนหน้า

### การวิเคราะห์ด้วยกราฟคิเวียต (Kiviat Graph Analysis)
เพื่อให้เห็นภาพรวมของระบบที่มีตัวแปรหลายด้านพร้อมกัน นักวิเคราะห์จึงนิยมใช้ **กราฟคิเวียต (Kiviat Graph)** ซึ่งเป็นเครื่องมือในการเปรียบเทียบเชิงภาพ (Pictorial Comparison) โดยมีหลักการอ่านข้อมูลดังนี้:

* **โครงสร้างกราฟ:** กราฟจะประกอบด้วยเส้นแกนที่ลากออกจากจุดศูนย์กลางไปยังเส้นรอบวงตามมุมที่เท่ากัน โดยแต่ละแกนจะแทนตัวแปรหนึ่งด้าน (เช่น ความเร็วในการอ่าน, ความเร็วในการเขียน, ราคา, ความจุ)
* **การตีความค่า:** จุดที่อยู่บนแกนหากยิ่งเข้าใกล้เส้นรอบวง (Circumference) มากเท่าใด จะหมายถึงระบบนั้นมี "ค่าที่ดีกว่า" ในด้านนั้นๆ
* **พื้นที่ครอบคลุม (Enclosed Area):** เมื่อลากเส้นเชื่อมต่อจุดในทุกแกนจะเกิดเป็นรูปทรงเฉพาะตัว หากระบบใดมี "พื้นที่ในรูปทรง" มากกว่า จะสื่อถึงการเป็นระบบที่ "ดีกว่า" หรือมีประสิทธิภาพโดยรวมสูงกว่าในภาพรวม

## บทสรุปเชิงเปรียบเทียบ (NOR vs. NAND)

แม้ว่าค่าทางเทคนิคเชิงปริมาณจะมีการเปลี่ยนแปลงและพัฒนาขึ้นในทุกปีตามความก้าวหน้าของเทคโนโลยีการผลิต แต่ความแตกต่างเชิงคุณภาพ (Relative Differences) ระหว่าง NOR และ NAND ยังคงมีความเสถียรและชัดเจนดังนี้:

| คุณลักษณะ | NOR Flash | NAND Flash |
| :--- | :--- | :--- |
| **การเข้าถึงข้อมูล** | ระดับบิต (Random Access) | ระดับบล็อก (Sequential Access) |
| **ความเร็วในการอ่าน** | สูงมาก | ปานกลาง |
| **ความเร็วในการเขียน/ลบ** | ต่ำ | สูง |
| **ความหนาแน่นของข้อมูล** | ต่ำ (ใช้พื้นที่มากต่อบิต) | สูง (ประหยัดพื้นที่) |
| **การประยุกต์ใช้งานหลัก** | เก็บ Code โปรแกรม (BIOS, Firmware) | เก็บข้อมูลจำนวนมาก (SSD, Flash Drive) |

# 5.5 เทคโนโลยีหน่วยความจำโซลิดสเตตแบบไม่ลบเลือนรุ่นใหม่

ในหัวข้อนี้ได้กล่าวถึงโครงสร้างลำดับชั้นของหน่วยความจำแบบดั้งเดิม (Traditional Memory Hierarchy) ซึ่งประกอบด้วย 3 ระดับหลัก เพื่อใช้เป็นเกณฑ์ในการเปรียบเทียบกับเทคโนโลยีใหม่ๆ ดังนี้:

### ลำดับชั้นของหน่วยความจำแบบดั้งเดิม (Traditional Memory Hierarchy)

* **Static RAM (SRAM):** เป็นหน่วยความจำที่โดดเด่นในด้านความเร็วในการเข้าถึงข้อมูลที่รวดเร็วที่สุด อย่างไรก็ตาม SRAM มีข้อจำกัดสำคัญคือมีต้นทุนการผลิตสูงที่สุด และมีความหนาแน่นของข้อมูล (Bit Density) ต่ำที่สุด ด้วยคุณสมบัติเหล่านี้จึงทำให้ SRAM เหมาะสำหรับการใช้งานเป็น **หน่วยความจำแคช (Cache Memory)**
    
* **Dynamic RAM (DRAM):** มีคุณลักษณะที่อยู่กึ่งกลาง โดยมีราคาถูกกว่าและมีความหนาแน่นของข้อมูลสูงกว่า SRAM แต่จะมีความเร็วในการทำงานที่ช้ากว่า โดยทั่วไป DRAM จึงเป็นตัวเลือกหลักในการนำมาใช้เป็น **หน่วยความจำหลักภายนอกชิป (Off-chip Main Memory)**
    
* **ฮาร์ดดิสก์ (Hard Disk):** อาศัยเทคโนโลยีจานแม่เหล็กในการจัดเก็บข้อมูล ซึ่งให้ความหนาแน่นของข้อมูลสูงมากและมีต้นทุนต่อบิตที่ต่ำที่สุด แต่ต้องแลกมาด้วยความเร็วในการเข้าถึงข้อมูลที่ค่อนข้างช้าเมื่อเทียบกับหน่วยความจำประเภทอื่น ฮาร์ดดิสก์จึงถูกจัดอยู่ในกลุ่ม **หน่วยความจำสำรองภายนอก (External Storage)** ภายใต้ลำดับชั้นของหน่วยความจำ

---

# นวัตกรรมโซลิดสเตตเมมโมรี่ประสิทธิภาพสูง

ในระบบลำดับชั้นของหน่วยความจำที่กล่าวมาข้างต้น **หน่วยความจำแฟลช (Flash Memory)** ได้เข้ามามีบทบาทสำคัญและถูกเพิ่มเข้าไปในโครงสร้างนี้ โดยข้อได้เปรียบที่สำคัญที่สุดของหน่วยความจำแฟลชเมื่อเปรียบเทียบกับหน่วยความจำแบบดั้งเดิม คือคุณสมบัติการเป็น **หน่วยความจำแบบไม่ลบเลือน (Nonvolatile)** ซึ่งสามารถรักษาข้อมูลไว้ได้แม้ไม่มีการจ่ายกระแสไฟฟ้า

### การประยุกต์ใช้งานตามลักษณะเฉพาะ
เทคโนโลยีแฟลชแต่ละประเภทถูกออกแบบมาเพื่อรองรับความต้องการที่แตกต่างกัน ดังนี้:
* **NOR Flash:** มีคุณสมบัติที่เหมาะสมที่สุดสำหรับการจัดเก็บโปรแกรม (Programs) และข้อมูลแอปพลิเคชันที่มีลักษณะคงที่ (Static Application Data) โดยเฉพาะในกลุ่ม **ระบบฝังตัว (Embedded Systems)**
* **NAND Flash:** มีลักษณะเด่นและประสิทธิภาพที่อยู่กึ่งกลางระหว่างหน่วยความจำหลักอย่าง DRAM และอุปกรณ์เก็บข้อมูลสำรองอย่าง ฮาร์ดดิสก์ (Hard disks) ทำให้มีความยืดหยุ่นในการใช้งานด้านการจัดเก็บข้อมูลปริมาณมาก

### วิวัฒนาการและข้อจำกัดในอนาคต
ที่ผ่านมา เทคโนโลยีหน่วยความจำทุกประเภทข้างต้นได้มีการพัฒนาในด้าน **การขยายขนาด (Scaling)** อย่างต่อเนื่อง เพื่อให้บรรลุเป้าหมายหลัก 4 ประการ ได้แก่:
1. การเพิ่มความหนาแน่นของข้อมูล (Higher bit density)
2. การเพิ่มความเร็วในการทำงาน (Higher speed)
3. การลดปริมาณการใช้พลังงาน (Lower power consumption)
4. การลดต้นทุนการผลิต (Lower cost)

อย่างไรก็ตาม สำหรับหน่วยความจำประเภทสารกึ่งตัวนำ (Semiconductor memory) นั้น ข้อมูลทางสถิติและแนวโน้มจาก ITRS (International Technology Roadmap for Semiconductors) ระบุว่า การรักษาอัตราการพัฒนาและปรับปรุงให้ก้าวหน้าในระดับเดิมนั้นเริ่มทำได้ยากลำบากมากขึ้นเรื่อยๆ เนื่องจากข้อจำกัดทางกายภาพและเทคโนโลยีการผลิตในปัจจุบัน

# เทคโนโลยีหน่วยความจำไม่ลบเลือนยุคใหม่ (Emerging Technologies)

จากการที่เทคโนโลยีสารกึ่งตัวนำแบบดั้งเดิมเริ่มเผชิญกับข้อจำกัดในการขยายขนาด (Scaling) จึงได้มีการพัฒนาหน่วยความจำไม่ลบเลือนรูปแบบใหม่ขึ้นมาเพื่อก้าวข้ามขีดจำกัดดังกล่าว โดยเทคโนโลยีที่มีความโดดเด่นและมีศักยภาพสูงสุดในปัจจุบันมีอยู่ 3 ประเภทหลัก ได้แก่:

1.  **Spin-transfer torque RAM (STT-RAM)**
2.  **Phase-change RAM (PCRAM)**
3.  **Resistive RAM (ReRAM)**

### สถานะการพัฒนาและการประยุกต์ใช้งานในปัจจุบัน
แม้ว่าเทคโนโลยีทั้งสามประเภทจะเข้าสู่กระบวนการผลิตในปริมาณมาก (Volume Production) แล้วก็ตาม แต่ในปัจจุบันยังคงถูกจำกัดการใช้งานอยู่ในกลุ่ม **แอปพลิเคชันเฉพาะทาง (Specialty Applications)** เท่านั้น เนื่องจากหน่วยความจำแบบ NAND Flash (และ NOR Flash ในบางส่วน) ยังคงครองตลาดหลักส่วนใหญ่ไว้ได้ ส่งผลให้เทคโนโลยีเกิดใหม่เหล่านี้ยังไม่สามารถก้าวขึ้นมาเป็นหน่วยความจำหลักที่มีความหนาแน่นสูง (Mainstream High-density) ได้ตามเป้าหมายที่ตั้งไว้ในตอนต้น อย่างไรก็ตาม คาดการณ์ว่าสถานการณ์ดังกล่าวมีแนวโน้มที่จะเปลี่ยนแปลงไปในอีกไม่กี่ปีข้างหน้า

---

### ตำแหน่งในลำดับชั้นของหน่วยความจำ (Memory Hierarchy)

จากแผนภาพลำดับชั้นของหน่วยความจำ (Figure 5.18) เทคโนโลยีใหม่เหล่านี้จะเข้ามาเติมเต็มช่องว่างระหว่างหน่วยความจำหลัก (DRAM) และหน่วยความจำสำรอง (NAND Flash/Hard Disk) โดยมีลักษณะการจัดวางตามประสิทธิภาพและต้นทุนดังนี้:


* **ทิศทางการเพิ่มประสิทธิภาพและความทนทาน (Increasing Performance and Endurance):**
    * **STT-RAM** จะถูกวางไว้ในระดับที่ใกล้เคียงกับ SRAM และ DRAM เนื่องจากมีความเร็วและความทนทานสูงที่สุดในกลุ่มเทคโนโลยีใหม่
    * **PCRAM** และ **ReRAM** จะอยู่ในระดับถัดลงมา โดยทำหน้าที่เป็นตัวเชื่อมระหว่างหน่วยความจำหลักและหน่วยความจำเก็บข้อมูล

* **ทิศทางการลดต้นทุนและเพิ่มความจุ (Decreasing Cost per bit, Increasing Capacity):**
    * ในขณะที่ไล่ลำดับลงมาสู่ **PCRAM**, **ReRAM** ไปจนถึง **NAND Flash** และ **Hard Disk** ต้นทุนต่อบิตจะลดต่ำลงเรื่อยๆ ในขณะที่ความสามารถในการจัดเก็บข้อมูล (Density) จะเพิ่มสูงขึ้น

# เทคโนโลยีหน่วยความจำ STT-RAM

**STT-RAM (Spin-Transfer Torque RAM)** เป็นหน่วยความจำประเภทแม่เหล็ก (Magnetic RAM หรือ MRAM) รูปแบบใหม่ที่มีคุณสมบัติโดดเด่นหลายประการ ได้แก่ การเป็นหน่วยความจำไม่ลบเลือน (Non-volatility), ความเร็วในการเขียนและอ่านที่สูงมาก (ประมาณ 6 ถึง 10 นาโนวินาที), ความทนทานในการบันทึกข้อมูลซ้ำได้มหาศาล (สูงถึง $7 \times 10^{15}$ รอบ) และไม่มีการใช้พลังงานในขณะสแตนด์บาย (Zero standby power)

### โครงสร้างและกลไกการกักเก็บข้อมูล
หัวใจสำคัญของความสามารถในการจัดเก็บข้อมูลของ STT-RAM คือโครงสร้างที่เรียกว่า **Magnetic Tunneling Junction (MTJ)** ซึ่งประกอบด้วยชั้นวัสดุแม่เหล็กเฟอร์โร (Ferromagnetic layers) สองชั้น ประกบบนและล่างของชั้นไดอิเล็กตริกบางๆ (Tunneling dielectric) ดังนี้:

* **ชั้นอ้างอิง (Pinned or Reference Layer):** เป็นชั้นแม่เหล็กที่ถูกออกแบบให้ทิศทางของอำนาจแม่เหล็กคงที่ ไม่เปลี่ยนแปลง
* **ชั้นอิสระ (Free Layer):** เป็นชั้นแม่เหล็กที่ทิศทางของอำนาจแม่เหล็กสามารถถูกปรับเปลี่ยนหรือ "พลิก" (Flip) ได้ผ่านกระบวนการเขียนข้อมูล

### สถานะความต้านทานและค่าทางตรรกะ
ความแตกต่างของทิศทางแม่เหล็กระหว่างชั้นอิสระและชั้นอ้างอิงจะส่งผลต่อค่าความต้านทานไฟฟ้าของ MTJ ซึ่งใช้แทนค่าข้อมูลทางตรรกะ:
* **ความต้านทานต่ำ (Low Resistance):** เกิดขึ้นเมื่อทิศทางแม่เหล็กของชั้นอิสระและชั้นอ้างอิงอยู่ในทิศทาง **ขนานกัน (Parallel)**
* **ความต้านทานสูง (High Resistance):** เกิดขึ้นเมื่อทิศทางแม่เหล็กของทั้งสองชั้นอยู่ในทิศทาง **ตรงกันข้ามกัน (Anti-parallel)**

### วิวัฒนาการจาก MRAM สู่ STT-RAM
ความแตกต่างที่สำคัญระหว่าง MRAM ยุคแรกกับ STT-RAM อยู่ที่กลไกในการเขียนข้อมูล (Write Mechanism):
* **MRAM รุ่นแรก:** ใช้สนามแม่เหล็กที่เหนี่ยวนำจากกระแสไฟฟ้า (Current-induced magnetic field) ในการเปลี่ยนทิศทางแม่เหล็กของชั้นอิสระ
* **STT-RAM:** ใช้กลไกใหม่ที่เรียกว่า **Polarization-current-induced magnetization switching** โดยทิศทางแม่เหล็กของชั้นอิสระจะถูกพลิกด้วยกระแสไฟฟ้าโดยตรง (Electrical current) 



### ข้อดีด้านการขยายขนาด (Scaling Property)
STT-RAM มีคุณสมบัติในการขยายขนาด (Scaling) ที่ดีกว่า MRAM รุ่นแรก เนื่องจากกระแสไฟฟ้าที่จำเป็นในการเปลี่ยนสถานะความต้านทานของ MTJ จะแปรผันตามขนาดพื้นที่ของเซลล์ MTJ (Proportional to cell area) ยิ่งเซลล์มีขนาดเล็กลง กระแสที่ใช้ก็จะยิ่งน้อยลงตามไปด้วย

# เทคโนโลยีหน่วยความจำ PCRAM

**PCRAM (Phase-change RAM)** ถือเป็นเทคโนโลยีที่มีความพร้อมและผ่านการพัฒนามาอย่างยาวนานที่สุด (Most mature) ในบรรดากลุ่มหน่วยความจำไม่ลบเลือนยุคใหม่ โดยมีเอกสารทางวิชาการและงานวิจัยรองรับอย่างกว้างขวาง

### พื้นฐานเทคโนโลยีและวัสดุ
เทคโนโลยี PCRAM มีพื้นฐานการทำงานมาจากวัสดุผสม **Chalcogenide Alloy** ซึ่งเป็นวัสดุประเภทเดียวกับที่ใช้ในสื่อบันทึกข้อมูลทางออปติคัล เช่น แผ่นคอมแพคดิสก์ (CD) และดีวีดี (DVD) 



### กลไกการกักเก็บข้อมูล (Data Storage Mechanism)
ความสามารถในการจัดเก็บข้อมูลของ PCRAM เกิดจากความแตกต่างของค่า **ความต้านทานไฟฟ้า (Resistance)** ระหว่าง 2 สถานะของวัสดุ Chalcogenide ดังนี้:
* **สถานะอะมอร์ฟัส (Amorphous Phase):** วัสดุจะมีโครงสร้างที่ไม่เป็นระเบียบ ส่งผลให้มี **ความต้านทานสูง (High-resistance)**
* **สถานะผลึก (Crystalline Phase):** วัสดุจะมีโครงสร้างที่เป็นระเบียบ ส่งผลให้มี **ความต้านทานต่ำ (Low-resistance)**

### กระบวนการเปลี่ยนสถานะ (SET and RESET Operations)
การเปลี่ยนสถานะของวัสดุเพื่อบันทึกข้อมูลทำได้โดยการใช้กระแสไฟฟ้าเข้าไปควบคุมอุณหภูมิภายในเซลล์หน่วยความจำ:
* **การปฏิบัติการ SET:** เป็นกระบวนการทำให้วัสดุกลายเป็นผลึก (Crystallized) โดยการจ่ายพัลส์ไฟฟ้า (Electrical pulse) เพื่อให้ความร้อนแก่เซลล์ในระดับที่สูงกว่าอุณหภูมิการเกิดผลึก (Crystallization temperature)
* **การปฏิบัติการ RESET:** เป็นกระบวนการทำให้วัสดุกลับสู่สถานะอะมอร์ฟัส โดยการจ่ายกระแสไฟฟ้าที่สูงกว่าเดิมเพื่อให้วัสดุหลอมละลาย (Melt) จากนั้นจึงทำการตัดกระแสไฟฟ้าอย่างรวดเร็ว (Quench) เพื่อให้วัสดุเย็นตัวลงทันทีจนไม่สามารถเรียงตัวเป็นผลึกได้

# เทคโนโลยีหน่วยความจำ ReRAM

**ReRAM** (หรือเรียกอีกชื่อหนึ่งว่า **RRAM**) เป็นเทคโนโลยีหน่วยความจำที่ทำงานโดยการสร้างสภาวะความต้านทานไฟฟ้า (Resistance) แทนที่จะเป็นการกักเก็บประจุไฟฟ้าโดยตรงเหมือนหน่วยความจำแบบดั้งเดิม

### กลไกการทำงานและหลักการจัดเก็บข้อมูล
กระบวนการบันทึกข้อมูลของ ReRAM อาศัยการจ่ายกระแสไฟฟ้าเข้าไปยังวัสดุเพื่อเปลี่ยนแปลงค่าความต้านทานภายในวัสดุนั้นๆ โดยสถานะความต้านทานที่ถูกเปลี่ยนไปจะสามารถนำมาวัดค่าเพื่อระบุผลลัพธ์เป็นค่าทางตรรกะ **1** หรือ **0** ได้ในขั้นตอนการอ่านข้อมูล



### สถานะการพัฒนาและคุณสมบัติทางเทคนิค
งานวิจัยและพัฒนา ReRAM ในปัจจุบันส่วนใหญ่ให้ความสำคัญกับสองประเด็นหลัก คือการค้นหาวัสดุที่เหมาะสมที่สุด (Appropriate materials) และการพัฒนาวิธีการวัดค่าสถานะความต้านทานของเซลล์หน่วยความจำให้มีความแม่นยำ โดย ReRAM มีคุณสมบัติเด่นทางวิศวกรรมดังนี้:
* **แรงดันไฟฟ้าต่ำ (Low Voltage):** การออกแบบวงจรทำงานภายใต้ระดับแรงดันไฟฟ้าที่น้อยกว่าเทคโนโลยีอื่น
* **ความทนทานสูง (High Endurance):** มีอายุการใช้งานและความทนทานที่เหนือกว่าหน่วยความจำแฟลช (Flash Memory) อย่างมาก
* **ขนาดเซลล์ที่เล็ก (Compact Cell Size):** ในเชิงทฤษฎี เซลล์ของ ReRAM มีขนาดที่เล็กกว่าเทคโนโลยีคู่แข่ง ซึ่งส่งผลดีต่อความหนาแน่นของข้อมูลในพื้นที่จำกัด

# สรุปภาพรวมเทคโนโลยีหน่วยความจำยุคใหม่ (Emerging Nonvolatile Memory)

เทคโนโลยีทั้งสามถูกพัฒนาขึ้นเพื่อก้าวข้ามขีดจำกัดในการขยายขนาด (Scaling) ของหน่วยความจำแฟลชแบบเดิม โดยมีจุดเด่นและกลไกการทำงานที่แตกต่างกัน ดังนี้:

### ตารางเปรียบเทียบคุณลักษณะทางเทคนิค

| คุณลักษณะ | **STT-RAM** | **PCRAM** | **ReRAM** |
| :--- | :--- | :--- | :--- |
| **กลไกหลัก** | การใช้ทิศทางแม่เหล็ก (MTJ) | การเปลี่ยนสถานะวัสดุ (Phase-change) | การสร้างความต้านทานไฟฟ้า |
| **จุดเด่นสำคัญ** | ความเร็วสูงมากและทนทานสูงสุด | เทคโนโลยีมีความพร้อม (Mature) ที่สุด | ขนาดเซลล์เล็กมากและใช้แรงดันไฟต่ำ |
| **ความเร็ว (Read/Write)** | **6 - 10 ns (เร็วที่สุด)** | ปานกลาง (เร็วกว่า Flash) | ปานกลาง (เร็วกว่า Flash) |
| **ความทนทาน (Endurance)** | **สูงมาก ($7 \times 10^{15}$ รอบ)** | สูงกว่า Flash | **สูงกว่า Flash อย่างมาก** |
| **พลังงานที่ใช้** | ต่ำมาก (Zero Standby Power) | ปานกลาง | **ต่ำ (Low Voltage)** |

---

### บทวิเคราะห์จุดเด่นรายเทคโนโลยี

* **STT-RAM (Spin-transfer torque RAM):** เป็นเทคโนโลยีที่เน้นประสิทธิภาพสูงสุดในกลุ่ม โดยใช้การพลิกทิศทางแม่เหล็กด้วยกระแสไฟฟ้าโดยตรง จุดเด่นคือความเร็วที่ใกล้เคียงกับ DRAM และความทนทานที่มหาศาล ทำให้เป็นตัวเลือกที่ดีที่สุดสำหรับการนำไปใช้เป็น **หน่วยความจำแคช (Cache)** หรือหน่วยความจำหลักที่ต้องเขียนข้อมูลบ่อยครั้ง
    
* **PCRAM (Phase-change RAM):** เป็นเทคโนโลยีที่ใช้การเปลี่ยนสถานะของวัสดุระหว่างผลึก (ความต้านทานต่ำ) และอะมอร์ฟัส (ความต้านทานสูง) เป็นเทคโนโลยีที่มีงานวิจัยรองรับมากที่สุดและมีความพร้อมในเชิงพาณิชย์สูง เหมาะสำหรับการนำมา **เสริมหรือทดแทน DRAM** ในส่วนของหน่วยความจำหลัก

* **ReRAM (Resistive RAM):** เน้นไปที่ความเรียบง่ายของโครงสร้างและขนาดที่เล็กมากในเชิงทฤษฎี ทำงานโดยการเปลี่ยนค่าความต้านทานของวัสดุด้วยกระแสไฟฟ้า มีจุดแข็งเรื่องการประหยัดพลังงานและความทนทานที่เหนือกว่า Flash Memory จึงมีศักยภาพสูงที่จะเข้ามาแทนที่ได้ทั้ง **หน่วยความจำหลักและหน่วยความจำสำรอง**

### บทสรุปในลำดับชั้นของหน่วยความจำ (Memory Hierarchy)

ในภาพรวม เทคโนโลยีเหล่านี้กำลังเข้ามาเติมเต็มช่องว่าง (Gap) ระหว่างหน่วยความจำที่เร็วแต่ลบเลือนได้ (SRAM/DRAM) กับหน่วยความจำที่ช้าแต่ไม่ลบเลือน (NAND Flash/Hard Disk) 

* **STT-RAM** จะขยับขึ้นไปแทนที่ในส่วนบนของลำดับชั้นเนื่องจากความเร็วสูง
* **PCRAM และ ReRAM** จะทำหน้าที่เป็นตัวเชื่อมโยงในระดับกลางที่เน้นความจุและความคุ้มค่า

แม้ในปัจจุบันเทคโนโลยีเหล่านี้จะยังถูกใช้งานในแอปพลิเคชันเฉพาะทาง (Specialty Applications) เนื่องจากการครอบครองตลาดของ NAND Flash แต่ด้วยข้อดีด้านการขยายขนาดและความทนทาน คาดว่าเทคโนโลยีเหล่านี้จะก้าวขึ้นมาเป็นมาตรฐานหลัก (Mainstream) ในอนาคตอันใกล้

# 5.6 คำศัพท์สำคัญ และคำถามทบทวน
**(Key Terms and Review Questions)**

### 1. คำศัพท์สำคัญ (Key Terms)
การทำความเข้าใจนิยามของคำศัพท์เหล่านี้จะช่วยให้สื่อสารและทำความเข้าใจเนื้อหาขั้นสูงได้แม่นยำขึ้น:

* **Bank Group:** กลุ่มของหน่วยความจำย่อยที่ช่วยให้สามารถรับ-ส่งข้อมูลได้พร้อมกันหลายทาง
* **Double Data Rate DRAM (DDR DRAM):** หน่วยความจำที่รับส่งข้อมูลได้ทั้งขาขึ้นและขาลงของสัญญาณนาฬิกา
* **Dynamic RAM (DRAM):** แรมที่ต้องคอยประจุไฟใหม่ตลอดเวลา (Refresh) มีความหนาแน่นสูง
* **Static RAM (SRAM):** แรมที่คงสถานะข้อมูลไว้ได้ตราบที่มีไฟเลี้ยงโดยไม่ต้อง Refresh มีความเร็วสูง
* **EEPROM:** หน่วยความจำถาวรที่ลบและเขียนซ้ำได้ด้วยสัญญาณไฟฟ้า
* **EPROM:** หน่วยความจำถาวรที่ลบข้อมูลได้ด้วยการส่องแสงอัลตราไวโอเลต (UV)
* **Error Correcting Code (ECC):** รหัสที่ใช้ตรวจจับและแก้ไขความผิดพลาดของบิตข้อมูล
* **Hamming Code:** อัลกอริทึมยอดนิยมในการระบุตำแหน่งและแก้ไขบิตที่ผิดพลาด
* **Flash Memory:** หน่วยความจำไม่ลบเลือนที่พัฒนามาจาก EEPROM ลบข้อมูลได้รวดเร็วเป็นบล็อก
* **MRAM / STT-RAM / PCRAM / ReRAM:** กลุ่มเทคโนโลยีหน่วยความจำอุบัติใหม่ที่มีประสิทธิภาพสูงและไม่ลบเลือนข้อมูล
* **Volatile Memory:** หน่วยความจำแบบลบเลือน (ข้อมูลหายเมื่อไม่มีไฟ)
* **Nonvolatile Memory:** หน่วยความจำแบบไม่ลบเลือน (ข้อมูลยังอยู่แม้ไม่มีไฟ)
* **Hard Failure:** ความเสียหายถาวรของฮาร์ดแวร์
* **Soft Error:** ความผิดพลาดชั่วคราวของข้อมูลที่เกิดจากสัญญาณรบกวน

### 2. แนวทางคำตอบสำหรับคำถามทบทวน (Review Questions)

**5.1 คุณสมบัติสำคัญของหน่วยความจำสารกึ่งตัวนำคืออะไร?**
* **ตอบ:** มีความสามารถในการแสดงสถานะทางไฟฟ้าที่แตกต่างกันสองสถานะ (เพื่อแทน 0 และ 1), สามารถถูกเขียนข้อมูลลงไปได้อย่างน้อยหนึ่งครั้งเพื่อกำหนดสถานะ และสามารถอ่านค่าเพื่อตรวจสอบสถานะที่บันทึกไว้ได้

**5.2 การตีความคำว่า "Random-Access Memory (RAM)" มี 2 แบบอย่างไร?**
* **ตอบ:** 1. ในเชิงกลไกการเข้าถึง: หมายถึงหน่วยความจำที่ใช้เวลาในการเข้าถึงข้อมูลทุกตำแหน่งเท่ากัน 2. ในเชิงการใช้งาน: หมายถึงหน่วยความจำหลักที่สามารถอ่านและเขียนข้อมูลใหม่ได้ตลอดเวลาในขณะที่เครื่องทำงาน

**5.3 & 5.4 ความแตกต่างระหว่าง DRAM และ SRAM (ด้านการใช้งานและคุณลักษณะ)?**
* **DRAM:** นิยมใช้เป็น **หน่วยความจำหลัก (Main Memory)** เนื่องจากมีขนาดเซลล์ที่เล็กมาก (ประหยัดพื้นที่) และราคาถูก แต่มีความเร็วต่ำกว่า
* **SRAM:** นิยมใช้เป็น **หน่วยความจำแคช (Cache)** เนื่องจากมีความเร็วในการเข้าถึงข้อมูลสูงมาก แต่มีราคาสูงและขนาดเซลล์ใหญ่กว่า

**5.5 ทำไม RAM ประเภทหนึ่งถึงถูกมองว่าเป็น Analog และอีกประเภทเป็น Digital?**
* **ตอบ:** **DRAM** ถูกมองว่าเป็นแอนาล็อกเพราะอาศัยการจัดเก็บ "ประจุไฟฟ้า" ในตัวเก็บประจุซึ่งสามารถรั่วไหลได้ตามเวลา ส่วน **SRAM** เป็นดิจิทัลเพราะใช้โครงสร้างลอจิกเกต (Flip-flop) เพื่อรักษาสถานะแรงดันไฟฟ้าให้คงที่

**5.6 ตัวอย่างการใช้งาน ROM มีอะไรบ้าง?**
* **ตอบ:** ใช้จัดเก็บโปรแกรมเริ่มต้นระบบ (Library subroutines for frequently wanted functions), ระบบควบคุมการทำงาน (System programs/BIOS) หรือตารางฟังก์ชันคงที่ต่างๆ

**5.7 ความแตกต่างระหว่าง EPROM, EEPROM และ Flash Memory?**
* **EPROM:** ลบข้อมูลด้วยแสง UV และต้องลบข้อมูลพร้อมกันทั้งชิป
* **EEPROM:** ลบด้วยไฟฟ้าและเลือกโหมดการลบในระดับไบต์ได้ แต่มีความหนาแน่นต่ำ
* **Flash Memory:** ลบด้วยไฟฟ้าได้รวดเร็ว โดยลบข้อมูลเป็นบล็อก (Block-level) ทำให้มีความจุสูงกว่า EEPROM มาก

**5.9 Parity Bit คืออะไร?**
* **ตอบ:** บิตที่ถูกเพิ่มเข้าไปในชุดข้อมูลเพื่อใช้ในการตรวจสอบความถูกต้อง โดยจะนับจำนวนบิตที่เป็น "1" ให้เป็นเลขคู่หรือเลขคี่ตามที่กำหนดไว้ เพื่อตรวจจับว่ามีบิตใดบิตหนึ่งผิดพลาดไปหรือไม่

**5.10 Syndrome ใน Hamming Code แปลความหมายอย่างไร?**
* **ตอบ:** ค่า Syndrome คือผลลัพธ์จากการตรวจสอบ Parity หากค่าเป็น 0 แสดงว่าไม่มีข้อผิดพลาด แต่หากไม่เป็น 0 ค่าที่เป็นตัวเลขนั้นจะระบุถึง "ตำแหน่งของบิต" ที่เกิดความผิดพลาดขึ้นโดยตรง

**5.11 SDRAM ต่างจาก DRAM ทั่วไปอย่างไร?**
* **ตอบ:** SDRAM จะทำงานสัมพันธ์ (Synchronize) กับสัญญาณนาฬิกาของระบบโดยตรง ทำให้การรับส่งข้อมูลระหว่าง CPU และ RAM มีความพร้อมเพรียงและรวดเร็วกว่า DRAM ทั่วไปที่ทำงานแบบอซิงโครนัส

**5.12 DDR RAM คืออะไร?**
* **ตอบ:** เป็นวิวัฒนาการของ SDRAM ที่สามารถส่งข้อมูลได้ 2 ครั้งต่อหนึ่งรอบสัญญาณนาฬิกา (ส่งทั้งช่วงขาขึ้นและขาลง) ทำให้ได้อัตราการส่งข้อมูลเพิ่มขึ้นเป็น 2 เท่า

**5.13 ความแตกต่างระหว่าง NAND และ NOR Flash?**
* **NOR Flash:** เข้าถึงข้อมูลแบบสุ่มได้เร็ว เหมาะสำหรับเก็บโปรแกรมคำสั่ง (Code)
* **NAND Flash:** ความหนาแน่นสูงและเขียนข้อมูลเร็ว เข้าถึงข้อมูลเป็นบล็อก เหมาะสำหรับเก็บข้อมูลปริมาณมาก (Data Storage) เช่น SSD

**5.14 จงระบุเทคโนโลยีหน่วยความจำไม่ลบเลือนรุ่นใหม่มา 3 ประเภท?**
* **ตอบ:** 1. **STT-RAM:** ใช้หลักการหมุนของอิเล็กตรอน 2. **PCRAM:** ใช้การเปลี่ยนสถานะวัสดุ 3. **ReRAM:** ใช้การเปลี่ยนความต้านทานไฟฟ้า

# 5.6 โจทย์ปัญหาและแบบฝึกหัด (Problems)
**(แปลและเฉลยแนวทางคำตอบ)**

### โจทย์ข้อ 5.1: การจัดองค์กรของชิป (RAM vs ROM)
**โจทย์:** จงวิเคราะห์เหตุผลว่าทำไมตามธรรมเนียมแล้ว RAM มักถูกจัดองค์กรแบบ **1 บิตต่อ 1 ชิป** ในขณะที่ ROM มักถูกจัดองค์กรแบบ **หลายบิตต่อ 1 ชิป**
* **แนวทางคำตอบ:** * **RAM:** เนื่องจาก RAM มักถูกนำมาต่อรวมกันเพื่อสร้างหน่วยความจำขนาดใหญ่ (เช่น 8GB, 16GB) การออกแบบให้ 1 ชิปส่งข้อมูลออกมาเพียง 1 บิต ช่วยให้เราสามารถนำชิปหลายตัวมาขนานกันเพื่อเพิ่มความกว้างของข้อมูล (Word size) เช่น ใช้ชิป 8 ตัวเพื่อให้ได้ข้อมูล 1 ไบต์ (8 บิต) ซึ่งช่วยลดจำนวนเข็มสัญญาณ (Pins) บนชิปแต่ละตัวให้น้อยลง ประหยัดพื้นที่และลดความซับซ้อนของแผงวงจร
    * **ROM:** มักใช้เก็บโปรแกรมหรือข้อมูลที่มีขนาดแน่นอนและมักใช้งานในอุปกรณ์ขนาดเล็ก การจัดองค์กรแบบหลายบิต (เช่น 8 บิต/1 ไบต์ต่อชิป) ช่วยให้สามารถดึงข้อมูลออกมาใช้งานได้ทันทีจากชิปเพียงตัวเดียว ซึ่งประหยัดพื้นที่ในอุปกรณ์ฝังตัว (Embedded Systems)

---

### โจทย์ข้อ 5.2: การคำนวณเวลาการ Refresh ของ DRAM
**โจทย์:** พิจารณา DRAM ที่ต้องทำการ Refresh 64 ครั้งในทุกๆ 1 ms โดยการ Refresh แต่ละครั้งใช้เวลา 150 ns และ Memory Cycle ปกติใช้เวลา 250 ns จงหาว่า "เปอร์เซ็นต์ของเวลาการทำงานทั้งหมดที่เสียไปกับการ Refresh" คือเท่าใด?

**วิธีทำ:**
1.  **หาเวลาที่ใช้ในการ Refresh ทั้งหมดต่อ 1 ms:**
    $$64 \text{ ครั้ง} \times 150 \text{ ns/ครั้ง} = 9,600 \text{ ns}$$
2.  **เทียบกับเวลาทั้งหมด (1 ms):**
    $$1 \text{ ms} = 1,000,000 \text{ ns}$$
3.  **คำนวณเป็นเปอร์เซ็นต์:**
    $$\frac{9,600 \text{ ns}}{1,000,000 \text{ ns}} \times 100 = 0.96\%$$
* **คำตอบ:** เสียเวลาไปกับการ Refresh เพียง **0.96%**

---

### โจทย์ข้อ 5.3: Timing Diagram และอัตราการส่งข้อมูล (Data Rate)
**โจทย์:** จากแผนผังเวลาการอ่านข้อมูล (Timing Diagram) ของ DRAM:
* **ก.** หาก Access time ($t1$ ถึง $t2$) คือ 60 ns และ Recharge time ($t2$ ถึง $t3$) คือ 40 ns จงหาค่า **Memory Cycle Time** และ **Maximum Data Rate** (เมื่อส่งออกครั้งละ 1 บิต)
* **ข.** หากสร้างระบบหน่วยความจำกว้าง 32 บิต จะได้อัตราการส่งข้อมูลเท่าใด?



**วิธีทำ:**
* **ข้อ ก.**
    1.  **Memory Cycle Time** = Access time + Recharge time
        $$60 \text{ ns} + 40 \text{ ns} = 100 \text{ ns}$$
    2.  **Data Rate (1-bit output)**: ใน 1 วินาที ($10^9 \text{ ns}$) จะทำงานได้กี่รอบ
        $$\frac{1}{100 \text{ ns}} = 10,000,000 \text{ รอบต่อวินาที (10 Mbps)}$$
* **ข้อ ข.**
    1.  **Data Transfer Rate (32-bit)**:
        $$10,000,000 \text{ รอบ} \times 32 \text{ บิต} = 320,000,000 \text{ bps (320 Mbps)}$$

---

### โจทย์ข้อ 5.4: การออกแบบโมดูลหน่วยความจำ (Memory Module Design)
**โจทย์:** จงวาดแผนผัง (High-level diagram) การสร้างหน่วยความจำขนาด **8 MB** โดยใช้ชิปขนาด **1 MB** จำนวน 8 ตัว (กำหนดให้ Word size คือ 1 ไบต์) พร้อมระบุการใช้เส้นสัญญาณ Address (Address lines)

**แนวทางคำอธิบายแผนผัง:**
1.  **จำนวนบิต Address:** เพื่อเข้าถึงข้อมูล 8 MB ($2^3 \times 2^{20} = 2^{23}$ ตำแหน่ง) ต้องใช้เส้นสัญญาณ Address ทั้งหมด **23 เส้น** (A0 - A22)
2.  **การแบ่งสัญญาณ:**
    * **A0 - A19 (20 เส้น):** ส่งไปยังชิป 1 MB ทุกตัวพร้อมกัน เพื่อเลือกตำแหน่งข้อมูลภายในชิปแต่ละตัว ($2^{20} = 1 \text{ MB}$)
    * **A20 - A22 (3 เส้น):** ส่งไปยังตัวถอดรหัส (**3-to-8 Decoder**) เพื่อเลือกเปิดการทำงานของชิปตัวใดตัวหนึ่ง (Chip Select - CS) จากทั้งหมด 8 ตัว
3.  **Data Lines:** เนื่องจากแต่ละชิปมีขนาด Word 1 ไบต์ (8 บิต) เส้นสัญญาณข้อมูล (D0 - D7) ของชิปทุกตัวจะถูกเชื่อมต่อขนานกันไปยัง Bus ข้อมูลหลัก

---

#### **ข้อ 5.5: การคำนวณ Access Time และ Wait State (ระบบ Intel 8086)**
**โจทย์:** ระบบทำงานที่ความถี่ 10 MHz โดย RAS ทำงานช้ากว่าสัญญาณ Address Enable 50 ns และต้องส่งข้อมูลให้ CPU ก่อนจบสถานะ T3 เป็นเวลา 60 ns
* **ก. Access Time ที่ต้องการ (แบบไม่มี Wait State):**
    1. หนึ่งรอบสัญญาณนาฬิกา (T) = $1 / 10 \text{ MHz} = 100 \text{ ns}$
    2. ช่วงเวลา T1 ถึง T3 รวมกันคือ $3 \times 100 = 300 \text{ ns}$
    3. RAS เริ่มทำงานที่กึ่งกลางของครึ่งหลังของ T1 + ดีเลย์ 50 ns: จะเริ่มที่ประมาณ $75 \text{ ns} + 50 \text{ ns} = 125 \text{ ns}$ นับจากจุดเริ่ม T1
    4. CPU อ่านข้อมูลที่ปลาย T3 แต่ข้อมูลต้องพร้อมก่อน 60 ns: คือที่เวลา $300 \text{ ns} - 60 \text{ ns} = 240 \text{ ns}$
    5. **Access Time** = เวลาที่ข้อมูลต้องพร้อม - เวลาที่เริ่ม RAS = $240 - 125 = 115 \text{ ns}$
* **ข. จำนวน Wait State (ถ้า Access Time เป็น 150 ns):**
    * หากใช้ 150 ns ข้อมูลจะพร้อมที่ $125 + 150 = 275 \text{ ns}$ ซึ่งเกินกำหนดที่ 240 ns ไป 35 ns จึงต้องเพิ่ม Wait State (รอบละ 100 ns) เข้าไป **1 รอบ (1 Wait State)**

#### **ข้อ 5.6: การ Refresh หน่วยความจำแบบคาบเวลา (Periodic Refresh)**
**โจทย์:** ใช้ DRAM ขนาด $64\text{K} \times 1$ มี 256 แถว ต้อง Refresh ทุกแถวภายใน 4 ms
* **ก. ระยะห่างระหว่างรอบ Refresh:**
    * $\frac{4 \text{ ms}}{256 \text{ แถว}} = 15.625 \mu\text{s}$ (หรือประมาณ **15.6 ไมโครวินาที**)
* **ข. ขนาดของตัวนับตำแหน่ง (Refresh Address Counter):**
    * เนื่องจากมี 256 แถว ($2^8$) จึงต้องใช้ตัวนับขนาด **8 บิต**

#### **ข้อ 5.7: การทำงานของ SRAM (Signetics 7489)**
* **หัวใจสำคัญ:** เป็นการวิเคราะห์สัญญาณ Chip Select (CS) และโหมดการทำงาน (Read/Write) ตามช่วงเวลาในแผนผัง
    * **ก. โหมด:** ขึ้นอยู่กับสัญญาณ Write Enable (WE) และ CS หาก CS ต่ำและ WE ต่ำจะเป็นโหมด "เขียน" หาก CS ต่ำและ WE สูงจะเป็นโหมด "อ่าน"
    * **ข. ข้อมูลตำแหน่ง 0-6:** ต้องดูค่าจาก Input Data ในจังหวะที่สัญญาณ Write ทำงาน
    * **ค. สถานะ Output:** ในจังหวะที่ไม่ได้อ่าน ข้อมูลขาออกมักจะเป็น High Impedance หรือสถานะพัก

#### **ข้อ 5.8: การออกแบบหน่วยความจำ 8192 บิต (16-bit word)**
**โจทย์:** ใช้ชิป SRAM $64 \times 1$ บิต สร้างหน่วยความจำรวม 8192 บิต
1. **จำนวนชิปที่ต้องใช้:** $8192 / 64 = 128 \text{ ชิป}$
2. **การจัดกลุ่ม:** เนื่องจากต้องการความกว้าง 16 บิต จึงต้องแบ่งชิปออกเป็นกลุ่มๆ ละ 16 ตัวขนานกัน (ได้ข้อมูล 16 บิตต่อชุด)
3. **ความจุรวมในเชิงตำแหน่ง:** $8192 / 16 \text{ บิต} = 512 \text{ ตำแหน่ง}$
4. **เส้นสัญญาณ:** ต้องใช้ Address 9 เส้น ($2^9 = 512$) เพื่อระบุตำแหน่ง

#### **ข้อ 5.9: การคำนวณ MTBF (Mean Time Between Failures)**
**โจทย์:** หน่วยความจำ 1 MB ($16\text{-bit processor}$) ใช้ชิป $256\text{K} \times 1$ โดยแต่ละชิปมีอัตราเสีย (FIT) 2000
1. **หาจำนวนชิปทั้งหมด:**
    * 1 MB คือ 8,388,608 บิต
    * ใช้ชิปขนาด 262,144 บิต ($256\text{K}$)
    * จำนวนชิป = $8,388,608 / 262,144 = 32 \text{ ชิป}$
2. **หาอัตราเสียรวม (Total FIT):**
    * $32 \text{ ชิป} \times 2000 \text{ FIT/ชิป} = 64,000 \text{ FIT}$ (เสีย 64,000 ครั้งต่อ $10^9$ ชั่วโมง)
3. **คำนวณค่า MTBF:**
    * $\text{MTBF} = \frac{10^9}{64,000} = 15,625 \text{ ชั่วโมง}$
* **สรุป:** โดยเฉลี่ยหน่วยความจำชุดนี้จะใช้งานได้ประมาณ **15,625 ชั่วโมง** ก่อนจะเกิดการเสียครั้งแรก

#### **ข้อ 5.10: เมื่อบิตตรวจสอบ (Check Bit) เกิดข้อผิดพลาดเอง**
**โจทย์:** จะเกิดอะไรขึ้นหากบิตที่ผิดพลาดคือบิตตรวจสอบ (Check Bit) ไม่ใช่บิตข้อมูล (Data Bit)?
* **คำตอบ:** อัลกอริทึมของ Hamming จะยังคงคำนวณค่า **Syndrome** ออกมาตามปกติ ซึ่งค่า Syndrome ที่ได้จะตรงกับ **เลขตำแหน่งของบิตตรวจสอบนั้นพอดี** (เช่น 1, 2, 4, 8) ระบบจะรับรู้ว่าบิตที่ตำแหน่งนั้นผิดพลาดและทำการสลับค่า (Flip) บิตนั้นกลับให้ถูกต้อง แต่เนื่องจากเป็นเพียงบิตตรวจสอบที่ผิด ข้อมูลจริง (Data Bits) จึงยังคงถูกต้องและใช้งานได้ตามเดิม

#### **ข้อ 5.11: การหา Check Bits สำหรับข้อมูล 8 บิต (11000010)**
**โจทย์:** จงหาบิตตรวจสอบที่จะถูกเก็บไว้พร้อมกับข้อมูล `11000010`
**วิธีทำ:**
1.  **วางตำแหน่งบิต:** ข้อมูล 8 บิตต้องการบิตตรวจสอบ 4 บิต (ตำแหน่ง 1, 2, 4, 8)
    
    * ตำแหน่ง: 1(C1), 2(C2), 3(D1), 4(C4), 5(D2), 6(D3), 7(D4), 8(C8), 9(D5), 10(D6), 11(D7), 12(D8)
    * วางข้อมูล: `_ _ 1 _ 1 0 0 _ 0 0 1 0`
2.  **คำนวณ Check Bits (ใช้ Even Parity):**
    * **C1** (ตรวจสอบตำแหน่ง 1,3,5,7,9,11): `_ 1 1 0 0 1` -> มี 1 สามตัว -> **C1 = 1**
    * **C2** (ตรวจสอบตำแหน่ง 2,3,6,7,10,11): `_ 1 0 0 0 1` -> มี 1 สองตัว -> **C2 = 0**
    * **C4** (ตรวจสอบตำแหน่ง 4,5,6,7,12): `_ 1 0 0 0` -> มี 1 หนึ่งตัว -> **C4 = 1**
    * **C8** (ตรวจสอบตำแหน่ง 8,9,10,11,12): `_ 0 0 1 0` -> มี 1 หนึ่งตัว -> **C8 = 1**
* **คำตอบ:** บิตตรวจสอบคือ **1011**

#### **ข้อ 5.12: การหาตำแหน่งบิตที่ผิดจากค่า Syndrome**
**โจทย์:** ข้อมูลเดิมมี Check Bits คือ `0111` แต่เมื่ออ่านออกมาคำนวณใหม่ได้ `1101` ข้อมูลที่อ่านได้คืออะไร?
**วิธีทำ:**
1.  หาค่า **Syndrome** โดยการ XOR ระหว่าง Check Bits เดิมและที่คำนวณใหม่:
    * `0111` XOR `1101` = `1010` (ฐานสอง)
2.  แปลงเป็นเลขฐานสิบ: $1010_2 = 10$
3.  **สรุป:** บิตที่ตำแหน่ง **10** เกิดข้อผิดพลาด ระบบจะทำการแก้ไขบิตที่ 10 ให้ถูกต้องก่อนส่งให้ผู้ใช้

#### **ข้อ 5.13: จำนวน Check Bits สำหรับข้อมูล 1024 บิต**
**โจทย์:** ต้องใช้กี่บิตเพื่อแก้ไข Single Bit Error ในข้อมูล 1024 บิต?
**วิธีทำ:** ใช้สูตร $2^K - 1 \ge M + K$ (โดย $M$ คือจำนวนบิตข้อมูล, $K$ คือบิตตรวจสอบ)
* ลองแทนค่า $K = 10$: $2^{10} - 1 = 1023$
    * $1023 \ge 1024 + 10$ (ไม่จริง: $1023 < 1034$)
* ลองแทนค่า $K = 11$: $2^{11} - 1 = 2047$
    * $2047 \ge 1024 + 11$ (จริง: $2047 \ge 1035$)
* **คำตอบ:** ต้องใช้บิตตรวจสอบ **11 บิต**

#### **ข้อ 5.14: การสร้าง SEC สำหรับข้อมูล 16 บิต**
**โจทย์:** สร้างรหัสสำหรับข้อมูล `0101000000111001` และพิสูจน์ว่าแก้ไขบิตที่ 5 ได้
**แนวคิด:**
1.  **บิตตรวจสอบ:** ข้อมูล 16 บิตต้องใช้บิตตรวจสอบ 5 บิต ($K=5$ เพราะ $2^5-1 = 31 \ge 16+5$)
2.  **การพิสูจน์:** เมื่อเกิด Error ใน Data Bit 5 (ซึ่งคือตำแหน่งที่ 9 ในแถว Hamming) ค่า Syndrome ที่คำนวณได้จากการเปรียบเทียบ Parity ของบิตตรวจสอบเดิมกับบิตตรวจสอบใหม่จะออกมาเป็น `01001` (หรือตำแหน่งที่ 9) ซึ่งจะชี้ไปยังตำแหน่งบิตที่ผิดอย่างแม่นยำ

# 5.1 SEMICONDUCTOR MAIN MEMORY  

## 1. Organization  
หน่วยความจำหลักในปัจจุบันใช้ Semiconductor โดยมีหน่วยพื้นฐานคือ Memory Cell  
Memory cell สามารถเขียน (Write) และอ่าน (Read) ได้ โดยเก็บข้อมูลเป็น 0 และ 1  

---

## 2. DRAM and SRAM  
Semiconductor Memory แบ่งได้ 2 ประเภทหลัก  

### 2.1 RAM (Volatile)  
สามารถอ่านและเขียนได้ แต่ข้อมูลจะหายเมื่อไฟดับ  

DRAM เก็บข้อมูลในรูปประจุไฟฟ้า ต้อง Refresh ตลอดเวลา  
ราคาถูก ความจุสูง → ใช้เป็น Main Memory  

SRAM ใช้วงจร Flip-Flop ไม่ต้อง Refresh  
เร็วกว่าแต่ราคาแพง → ใช้เป็น Cache  

### 2.2 ROM (Non-Volatile)  
เก็บข้อมูลถาวร เน้นอ่าน ไม่เหมาะกับการเขียน  

---

## 3. Types of ROM  
ROM : เขียนมาจากโรงงาน แก้ไม่ได้  
PROM : เขียนได้ครั้งเดียว  
EPROM : ลบด้วยแสง UV และเขียนใหม่ได้  
EEPROM : ลบ/เขียนใหม่ด้วยไฟฟ้า (ระดับ Byte)  
Flash Memory : ลบเร็วกว่า และเก็บข้อมูลหนาแน่นกว่า  

---

## 4. Chip Logic (Organization & Chip Logic)  
การจัดเรียง : Memory Cell เรียงเป็น Row/Column (1 cell = 1 bit)  
การเข้าถึงข้อมูล : ใช้ Address และสัญญาณควบคุม เช่น WE, OE  
DRAM ใช้ RAS และ CAS เพื่อเลือก Row และ Column  
DRAM ต้อง Refresh ตลอดเวลา  

---

## 5. The Trade-off  
ไม่มีหน่วยความจำที่ดีที่สุด ต้องแลกกันระหว่าง Speed, Price, Capacity  

---

# บทที่ 5.2 เรื่อง Error Correction (การแก้ไขข้อผิดพลาดในหน่วยความจำ) 
ได้อธิบายถึงระบบหน่วยความจำเซมิคอนดักเตอร์ที่มักจะพบข้อผิดพลาด โดยสามารถแบ่งชนิดของข้อผิดพลาดออกเป็น 2 ประเภทหลัก ได้แก่:

## 1. Hard failures (ข้อผิดพลาดแบบถาวร): 
เป็นความบกพร่องทางกายภาพที่ทำให้เซลล์หน่วยความจำไม่สามารถจัดเก็บข้อมูลได้อย่างเชื่อถือได้ และจะค้างอยู่ที่ค่า 0 หรือ 1 เสมอ หรือสลับค่าไปมาแบบควบคุมไม่ได้ ซึ่งอาจเกิดจากการใช้งานในสภาพแวดล้อมที่รุนแรง ความบกพร่องจากกระบวนการผลิต หรือการเสื่อมสภาพจากการใช้งาน

---

## 2. Soft errors (ข้อผิดพลาดแบบชั่วคราว): 
เป็นเหตุการณ์สุ่มที่ไม่ทำลายเซลล์หน่วยความจำ แต่ทำให้ค่าข้อมูลในหน่วยความจำเปลี่ยนไป สาเหตุมักเกิดจากปัญหาของแหล่งจ่ายไฟ หรืออนุภาคแอลฟา (Alpha particles) จากการสลายตัวของสารกัมมันตรังสีที่มีอยู่ในวัสดุเกือบทุกชนิด

**กระบวนการตรวจจับและแก้ไขข้อผิดพลาด (Error-Correcting Codes):** เพื่อจัดการกับปัญหาดังกล่าว ระบบหน่วยความจำหลักสมัยใหม่จึงมีลอจิกสำหรับคอยตรวจจับและแก้ไขข้อมูลโดยมีกระบวนการทำงานคือ:

* **ตอนเขียนข้อมูล:** ข้อมูลขนาด M บิต จะถูกนำไปคำนวณผ่านฟังก์ชัน เพื่อสร้าง "รหัส (Code)" ตรวจสอบขนาด K บิต ดังนั้นขนาดของข้อมูลทั้งหมดที่ถูกจัดเก็บจริงลงในหน่วยความจำคือ M+K บิต
* **ตอนอ่านข้อมูล:** ระบบจะสร้างรหัส K บิตชุดใหม่ขึ้นมาจากข้อมูลที่อ่าน แล้วนำไปเปรียบเทียบกับรหัส K บิตเดิมที่เคยดึงออกมา

ซึ่งผลลัพธ์ที่เป็นไปได้มี 3 กรณี:
1. ไม่พบข้อผิดพลาด ระบบจะส่งข้อมูลออกไปใช้งานได้เลย
2. พบข้อผิดพลาดและสามารถแก้ไขได้ ข้อมูลและรหัสจะถูกส่งเข้าไปในตัวแก้ไข (Corrector) เพื่อปรับข้อมูล M บิตให้ถูกต้องก่อนส่งออกไป
3. พบข้อผิดพลาดแต่ไม่สามารถแก้ไขได้ ระบบจะทำการรายงานความผิดปกตินี้

**รหัสแฮมมิง (Hamming Code):** หนึ่งในวิธีแก้ไขข้อผิดพลาดที่เข้าใจง่ายที่สุดคิดค้นโดย Richard Hamming ระบบนี้จะเปรียบเทียบรหัสเดิมกับรหัสใหม่แบบบิตต่อบิตด้วยการทำ **Exclusive-OR (XOR)** ซึ่งผลลัพธ์ที่ได้เรียกว่า **Syndrome word** ขนาด K บิต

* จำนวนของบิตตรวจสอบ (K) จะต้องสอดคล้องกับอสมการ: $2^K - 1 \ge M + K$
* ตัวอย่างเช่น หากข้อมูลมีขนาด 8 บิต (M=8) จะต้องใช้บิตตรวจสอบ 4 บิต (K=4) เป็นอย่างน้อย เนื่องจาก $2^4 - 1 \ge 8 + 4$ (หรือ $15 \ge 12$)

**ลักษณะการอ่านค่า Syndrome Word:**
* **ถ้าค่าเป็น 0 ทั้งหมด:** หมายถึงไม่พบข้อผิดพลาด
* **ถ้ามีเลข 1 เพียงบิตเดียว:** หมายความว่ามีข้อผิดพลาดเกิดขึ้นที่ตัวบิตตรวจสอบ (Check bits) เอง ซึ่งระบบไม่ต้องทำการแก้ไขข้อมูล
* **ถ้ามีเลข 1 มากกว่าหนึ่งบิต:** ค่าตัวเลขของ Syndrome จะระบุตำแหน่งของบิตข้อมูลที่ผิดพลาด เพื่อให้ระบบทำการกลับค่า (Invert) บิตในตำแหน่งนั้นให้กลับมาถูกต้อง โดยบิตตรวจสอบ (Check bits) จะถูกกำหนดให้อยู่ในตำแหน่งที่เป็นเลขยกกำลังของ 2 เสมอ (เช่น ตำแหน่งที่ 1, 2, 4, และ 8)

**ระบบ SEC-DED (Single-Error-Correcting, Double-Error-Detecting):** โดยทั่วไปแล้ว ในระบบหน่วยความจำมักจะนิยมติดตั้งรหัสที่เรียกว่า **SEC-DED** ซึ่งมีความสามารถในการ **"แก้ไขข้อผิดพลาดได้ 1 บิต และตรวจจับข้อผิดพลาดที่เกิดพร้อมกัน 2 บิตได้"**

เพื่อป้องกันการที่ระบบวิเคราะห์ผิดพลาดเมื่อมีข้อมูลเสีย 2 บิตพร้อมกัน SEC-DED จึงต้องเพิ่มความปลอดภัยโดยการใส่บิตตรวจสอบ (Parity bit) เข้าไปอีก 1 บิต ซึ่งการเพิ่มระบบแก้ไขข้อผิดพลาดเหล่านี้ทำให้หน่วยความจำมีความน่าเชื่อถือสูง แต่ก็ต้องแลกมากับความจุที่ต้องเตรียมเผื่อไว้จัดเก็บรหัส (Overhead) ตัวอย่างเช่น ในระบบ DRAM สมัยใหม่อาจต้องใช้พื้นที่สูญเปล่าเพิ่มขึ้นถึง 7% ถึง 20% เพื่อจัดเก็บรหัสป้องกันเหล่านี้

SRAM : เร็วมาก แต่แพง → Cache  
DRAM : ช้ากว่า แต่ถูกและจุเยอะ → Main Memory  
ROM : ใช้เก็บข้อมูลถาวร

# 5.4 หน่วยความจำแบบแฟลช (Flash Memory)

หน่วยความจำแบบแฟลช (Flash Memory) เป็นหนึ่งในนวัตกรรมสำคัญของหน่วยความจำสารกึ่งตัวนำ (Semiconductor Memory) ที่ได้รับการพัฒนาขึ้นในช่วงกลางทศวรรษที่ 1980 โดยมีบทบาทสำคัญทั้งในฐานะหน่วยความจำภายในเครื่องและหน่วยความจำภายนอก ความโดดเด่นของเทคโนโลยีนี้คือการเป็นจุดกึ่งกลางที่ลงตัวระหว่างหน่วยความจำประเภท EPROM และ EEPROM ทั้งในด้านต้นทุนการผลิตและประสิทธิภาพการใช้งาน

### ลักษณะการทำงานเบื้องต้น
* **เทคโนโลยีการลบข้อมูล:** ใช้การลบด้วยไฟฟ้า (Electrical Erasing) เช่นเดียวกับ EEPROM แต่มีความเร็วที่เหนือกว่ามาก โดยสามารถลบข้อมูลทั้งหมดในชิปได้ภายในเวลาเพียงไม่กี่วินาที
* **การลบข้อมูลแบบ "บล็อก" (Block):** สามารถเลือกการลบข้อมูลเป็นส่วนๆ หรือ "บล็อก" แทนที่จะต้องลบข้อมูลทั้งชิปพร้อมกัน ซึ่งเป็นที่มาของชื่อ "แฟลช" (Flash) เนื่องจากลักษณะการจัดระเบียบไมโครชิปที่ยอมให้มีการลบข้อมูลเป็นส่วนๆ ได้ในการกระทำเพียงครั้งเดียว
* **ข้อจำกัดและความคุ้มค่า:** ไม่สามารถลบข้อมูลในระดับ "ไบต์" (Byte) ได้โดยอิสระ แต่ด้วยโครงสร้างที่ใช้ทรานซิสเตอร์เพียงตัวเดียวต่อหนึ่งบิต ทำให้หน่วยความจำแฟลชมีความหนาแน่นของข้อมูลสูงและมีราคาประหยัดใกล้เคียงกับ EPROM

---

### กลไกการทำงานทางเทคนิค
การทำงานของเซลล์หน่วยความจำแฟลชมีพื้นฐานมาจากหลักการของทรานซิสเตอร์ทั่วไป ซึ่งอาศัยคุณสมบัติของสารกึ่งตัวนำในการควบคุมการไหลของกระแสไฟฟ้าจาก **"ซอร์ส" (Source)** ไปยัง **"เดรน" (Drain)** โดยใช้แรงดันไฟฟ้าจำนวนเล็กน้อยที่จ่ายให้แก่ **"เกต" (Gate)** เป็นตัวควบคุม



สิ่งที่ทำให้เซลล์หน่วยความจำแฟลชแตกต่างออกไปคือ:
* **เกตลอย (Floating Gate):** มีการเพิ่มส่วนนี้เข้าไปในโครงสร้าง โดยถูกหุ้มด้วยชั้นออกไซด์บางๆ เพื่อทำหน้าที่เป็นฉนวน
* **สถานะลอจิก "1":** ในสถานะเริ่มต้นที่เกตลอยไม่มีประจุไฟฟ้า เซลล์จะทำงานเสมือนทรานซิสเตอร์ปกติ
* **สถานะลอจิก "0":** เมื่อมีการจ่ายแรงดันไฟฟ้าที่สูงเพียงพอข้ามชั้นออกไซด์ จะเกิดปรากฏการณ์ที่อิเล็กตรอนสามารถเคลื่อนที่ผ่านชั้นฉนวนเข้าไปติดอยู่ในเกตลอยได้ หรือที่เรียกว่า **"การทะลุผ่าน" (Tunneling)** อิเล็กตรอนที่ถูกกักขังอยู่นี้จะยังคงอยู่แม้จะไม่มีกระแสไฟฟ้ามาเลี้ยงวงจร
* **การตรวจสอบและคืนสถานะ:** การตรวจสอบสถานะทำได้ผ่านวงจรภายนอกที่ทดสอบการนำไฟฟ้า และเมื่อต้องการลบข้อมูลหรือคืนสถานะกลับเป็น "1" ระบบจะใช้แรงดันไฟฟ้าสูงในทิศทางตรงกันข้ามเพื่อดึงอิเล็กตรอนออกจากเกตลอย

> **คุณสมบัติการคงสภาพข้อมูล (Persistence):** หน่วยความจำแฟลชเป็นหน่วยความจำแบบถาวร (Persistent Memory) ข้อมูลจะยังคงอยู่แม้ไม่มีกระแสไฟฟ้า จึงถูกนำไปใช้งานทั้งในฐานะหน่วยความจำสำรองภายนอก (Secondary Storage) และทางเลือกทดแทน RAM ในระบบคอมพิวเตอร์บางประเภท

### ประเภทของหน่วยความจำแฟลช: NOR และ NAND
เทคโนโลยีแฟลชสามารถจำแนกออกเป็น 2 ประเภทหลักตามลักษณะโครงสร้างทางตรรกะและการเชื่อมต่อ ดังนี้:

#### 1. หน่วยความจำแฟลชแบบ NOR (NOR Flash)
* **โครงสร้าง:** เซลล์แต่ละเซลล์จะถูกเชื่อมต่อแบบขนาน (Parallel) เข้ากับสายบิต (Bit lines)
* **การทำงาน:** สามารถอ่าน เขียน หรือลบข้อมูลในระดับบิตหรือไบต์ได้อย่างอิสระ มีลักษณะการทำงานคล้ายกับลอจิกเกตแบบ NOR
* **จุดเด่นและการใช้งาน:** มีความเร็วสูงในการเข้าถึงข้อมูลเฉพาะจุด (Random Access) นิยมใช้ในระบบฝังตัว (Embedded Systems) และไมโครคอนโทรลเลอร์ เพื่อจัดเก็บโปรแกรมคำสั่งที่มีขนาดไม่ใหญ่นักและข้อมูลที่ไม่ค่อยเปลี่ยนแปลงบ่อย

#### 2. หน่วยความจำแฟลชแบบ NAND (NAND Flash)
* **โครงสร้าง:** จัดระเบียบในรูปแบบของชุดทรานซิสเตอร์ที่เชื่อมต่อกันแบบอนุกรม (Series) โดยปกติจะมีทรานซิสเตอร์ 16 หรือ 32 ตัวต่อหนึ่งชุด ทำงานคล้ายลอจิกเกตแบบ NAND
* **จุดเด่น:** มีความหนาแน่นสูง (High Density) บรรจุข้อมูลได้มากในพื้นที่ขนาดเล็ก และมีความเร็วในการเขียนข้อมูลที่เหนือกว่าแบบ NOR
* **ข้อจำกัด:** ไม่มีบัสที่อยู่ภายนอกสำหรับการเข้าถึงแบบสุ่ม การอ่านข้อมูลจึงต้องทำเป็นส่วนๆ ที่เรียกว่า "หน้า" (Page) หรือ "บล็อก" (Block)
* **การใช้งาน:** เหมาะสำหรับเป็นหน่วยความจำภายนอก เช่น แฟลชไดรฟ์ USB, เมมโมรี่การ์ด และ SSD (Solid-State Disks)

### การเปรียบเทียบประสิทธิภาพด้วยกราฟคิเวียต (Kiviat Graphs)
แม้สถาปัตยกรรมทั้งสองจะมีการพัฒนาสมรรถนะในทุกปี แต่ความแตกต่างเชิงสัมพัทธ์ยังคงชัดเจน ซึ่งอธิบายได้ผ่าน **กราฟคิเวียต (Kiviat Graphs)** หรือกราฟใยแมงมุม:

* **หลักการอ่าน:** ยิ่งพื้นที่ที่ล้อมรอบด้วยเส้นกราฟมีขนาดกว้างมากเท่าใด (เข้าใกล้เส้นรอบวงมากขึ้น) แสดงว่าระบบนั้นมีประสิทธิภาพในภาพรวมดีขึ้นเท่านั้น
* **NOR Flash:** โดดเด่นในด้านความเร็วการอ่านและการเข้าถึงแบบสุ่ม
* **NAND Flash:** ชนะในด้านความหนาแน่นของข้อมูลและต้นทุนต่อหน่วยความจุ
